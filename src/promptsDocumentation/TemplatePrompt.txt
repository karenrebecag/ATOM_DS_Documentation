# Import Pattern Flexibility (Updated 2026-03-31)

All ATOM packages support **multiple import patterns** thanks to improved package.json exports configuration following Node.js best practices. This means AI tools and bundlers can generate imports in different styles, and they'll all work correctly.

## Supported Import Styles

### Tokens Package
```typescript
// Shorthand (recommended)
import '@atomchat.io/tokens/css'
import '@atomchat.io/tokens/scss'

// Full path (also supported)
import '@atomchat.io/tokens/build/css/tokens.css'
import '@atomchat.io/tokens/build/scss/_tokens.scss'
import '@atomchat.io/tokens/build/json/tokens.json'

// JavaScript/TypeScript
import tokens from '@atomchat.io/tokens'
// or with full path
import tokens from '@atomchat.io/tokens/build/js/tokens.js'
```

### CSS Package
```typescript
// Shorthand (recommended)
import '@atomchat.io/css'

// Alternative paths (all work)
import '@atomchat.io/css/atom.css'
import '@atomchat.io/css/dist/atom.css'
```

### Animations Package
```typescript
// Shorthand (recommended)
import { initBadge } from '@atomchat.io/animations/badge'
import { initAllAnimations } from '@atomchat.io/animations'

// Full path (also supported)
import { initBadge } from '@atomchat.io/animations/dist/badge.js'
import { initAllAnimations } from '@atomchat.io/animations/dist/index.js'
```

### Components Package (Astro)
```typescript
// Shorthand (recommended)
import Button from '@atomchat.io/components-astro/atoms/Button.astro'

// Full path (also supported)
import Button from '@atomchat.io/components-astro/src/atoms/Button.astro'
```

**Use whichever style your tooling prefers** — they're all valid and resolve correctly.

**⚠️ CRITICAL IMPORT ORDER — THIS IS NON-NEGOTIABLE:**

```tsx
// ✅ CORRECT ORDER (always use this exact order):
import '@atomchat.io/tokens/css';  // 1. FIRST
import '@atomchat.io/css';         // 2. SECOND
import './global.css';             // 3. THIRD (optional)

// ❌ WRONG ORDER (will break all styles):
import '@atomchat.io/css';         // ❌ CSS before tokens = BROKEN
import '@atomchat.io/tokens/css';  // ❌ Too late, var() already empty

// ❌ WRONG IMPORTS (forbidden):
import 'tailwindcss/tailwind.css'; // ❌ NO Tailwind
import './tailwind.css';           // ❌ NO Tailwind
```

**Why this order matters:**
- `@atomchat.io/tokens/css` defines CSS custom properties: `--bg-primary`, `--fg-default`, etc.
- `@atomchat.io/css` uses those custom properties: `background: var(--bg-primary);`
- If you import CSS before tokens, all `var()` references are empty → components render without styles
- **If components look unstyled, check import order FIRST**

**❌ DO NOT add these to the layout:**
```html
<!-- ❌ WRONG: NO Tailwind CDN -->
<link href="https://cdn.tailwindcss.com" rel="stylesheet" />

<!-- ❌ WRONG: NO Google Fonts (ATOM uses Boysen/Median) -->
<link href="https://fonts.googleapis.com/css2?family=Inter" rel="stylesheet" />

<!-- ❌ WRONG: NO external CSS frameworks -->
<link href="bootstrap.min.css" rel="stylesheet" />
```

### If animations are enabled, add this script to the layout:

**⚠️ COPY THIS CODE EXACTLY — REQUIRED FOR ANIMATIONS:**

```astro
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  // Initialize on page load
  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  // Re-initialize on Astro page transitions
  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**⚠️ CRITICAL: This script is REQUIRED for animations to work**
- Without this script, components will have no animations
- Button hover effects, badge pulses, spinner rotations will not work
- Data attributes like `data-hover-rotate` need this script to activate
- **If animations aren't working, check if this script exists**

**❌ DO NOT use these animation patterns:**
```css
/* ❌ WRONG: CSS-only animations */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
.button:hover { animation: fadeIn 0.3s; }

/* ❌ WRONG: Transition-only hover effects */
.button:hover { transform: translateY(-2px); }
```

```tsx
// ❌ WRONG: Framer Motion
import { motion } from 'framer-motion';
<motion.div animate={{ opacity: 1 }} />

// ❌ WRONG: React Spring
import { useSpring } from '@react-spring/web';

// ❌ WRONG: Custom useEffect animations
useEffect(() => {
  // ❌ Don't write custom animation code
}, []);
```

**✅ CORRECT: GSAP via ATOM animations**
```tsx
// ✅ Just call initAllAnimations() in the layout
import { initAllAnimations } from '@atomchat.io/animations';
initAllAnimations();

// ✅ Components automatically animate via data attributes
<Button data-hover-rotate>  {/* Auto-animated */}
<Badge data-pulse>           {/* Auto-animated */}
<Spinner data-spin>          {/* Auto-animated */}
```

## Theme Toggle (if Light + Dark selected)

Add theme toggle to your layout or navigation:
```astro
<button id="theme-toggle" aria-label="Toggle theme">
  Toggle Theme
</button>

<script>
  const toggle = document.getElementById('theme-toggle');
  const html = document.documentElement;

  // Check saved preference or system preference
  const saved = localStorage.getItem('theme');
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  const initial = saved || (prefersDark ? 'dark' : 'light');
  html.setAttribute('data-theme', initial);

  toggle?.addEventListener('click', () => {
    const current = html.getAttribute('data-theme');
    const next = current === 'light' ? 'dark' : 'light';
    html.setAttribute('data-theme', next);
    localStorage.setItem('theme', next);
  });
</script>
```

ATOM Design System is token-first. Every visual value comes from CSS custom properties.

## Token Categories

### Colors
```css
/* Backgrounds */
var(--bg-primary)          /* Main background */
var(--bg-secondary)        /* Secondary surfaces */
var(--bg-tertiary)         /* Tertiary surfaces */
var(--bg-inverse)          /* Inverted background */
var(--bg-disabled)         /* Disabled state */

/* Foreground / Text */
var(--fg-default)          /* Primary text */
var(--fg-secondary)        /* Secondary text */
var(--fg-tertiary)         /* Muted text */
var(--fg-inverse)          /* Inverted text */
var(--fg-disabled)         /* Disabled text */

/* Accent */
var(--fg-accent)           /* Brand accent color */
var(--bg-accent)           /* Accent background */
```

### Spacing
```css
var(--xxs)    /* 4px */
var(--xs)     /* 8px */
var(--s)      /* 12px */
var(--sm)     /* 14px */
var(--m)      /* 16px */
var(--l)      /* 24px */
var(--xl)     /* 32px */
var(--xxl)    /* 48px */
var(--xxxl)   /* 64px */
```

### Typography
```css
/* Font sizes */
var(--text-caption)         /* 12px */
var(--text-label-regular)   /* 14px */
var(--text-body-regular)    /* 16px */
var(--text-body-large)      /* 18px */
var(--text-heading-s)       /* 20px */
var(--text-heading-m)       /* 24px */
var(--text-heading-l)       /* 32px */
var(--text-heading-xl)      /* 40px */
var(--text-display)         /* 48px */

/* Font weights */
var(--font-weight-regular)  /* 400 */
var(--font-weight-medium)   /* 500 */
var(--font-weight-semibold) /* 600 */
var(--font-weight-bold)     /* 700 */
```

### Border & Radius
```css
var(--radius-s)    /* 4px */
var(--radius-m)    /* 8px */
var(--radius-l)    /* 12px */
var(--radius-xl)   /* 16px */
var(--radius-full) /* 9999px (pill) */

var(--stroke-xs)   /* 1px */
var(--stroke-s)    /* 2px */
```

### Shadows
```css
var(--shadow-xs)   /* Subtle elevation */
var(--shadow-s)    /* Card elevation */
var(--shadow-m)    /* Dropdown elevation */
var(--shadow-l)    /* Modal elevation */
```

## Critical Rule: NEVER Hardcode Values

```css
/* CORRECT */
.my-section {
  padding: var(--xl) var(--l);
  background: var(--bg-primary);
  color: var(--fg-default);
  border-radius: var(--radius-m);
}

/* WRONG — Never do this */
.my-section {
  padding: 32px 24px;
  background: #ffffff;
  color: #1a1a1a;
  border-radius: 8px;
}
```

Now that your project is configured and your first page is built, here's how to keep building:

## Adding New Sections
1. Use `<Section>` + `<Container>` + `<Stack>` for consistent layout
2. Compose UI with the 23 available ATOM components
3. Style custom elements exclusively with design tokens
4. Keep each section in its own Astro component for maintainability

```astro
    </Container>
  </Section>

```

## Exploring Components
Each ATOM component has its own documentation page with:
- Live previews and playgrounds
- Full props API
- Required design tokens
- Usage examples
- Accessibility specifications

Visit the component documentation to learn the complete API for each component before using it.

## Implementation Rules (Non-Negotiable)
1. **Token-first** — Every color, spacing, font, radius, and shadow must use `var(--token-name)`
2. **No hardcoded values** — If a token doesn't exist for what you need, ask before inventing one
3. **Semantic HTML** — Use correct elements (button for actions, a for links, etc.)
4. **Accessible** — All interactive elements must have ARIA labels, keyboard support, and focus states
5. **Responsive** — Mobile-first approach, test at 320px, 768px, 1024px, 1440px
6. **Performance** — Astro ships zero JS by default. Only add `client:load` when interactivity is required
7. **NO EMOJIS** — NEVER use emojis in UI unless explicitly requested. ALWAYS use Lucide React icons (`lucide-react`)

# Troubleshooting: Styles Not Loading

If components render without styles (plain HTML with no colors, spacing, or borders), verify the following:

## 1. Check CSS Custom Properties in Browser Console

```javascript
// Run in browser DevTools console
const root = getComputedStyle(document.documentElement);
console.log({
  '--bg-primary': root.getPropertyValue('--bg-primary'),
  '--fg-default': root.getPropertyValue('--fg-default'),
  '--l': root.getPropertyValue('--l')
});
```

**Expected result:** All values should return colors/sizes, NOT empty strings.

**If all values are empty (`""`):**
- ❌ Tokens CSS is missing or not loaded
- ✅ Fix: Import `@atomchat.io/tokens/css` BEFORE `@atomchat.io/css` in Layout.astro

## 2. Check Import Order in Layout

```astro
---
// ✅ CORRECT ORDER
import '@atomchat.io/tokens/css';  // FIRST: Define CSS custom properties
import '@atomchat.io/css';         // SECOND: Use those custom properties
import '../styles/global.css';     // THIRD: Your overrides

// ❌ WRONG ORDER
import '@atomchat.io/css';         // Component styles load before tokens are defined
import '@atomchat.io/tokens/css';  // Too late! CSS already tried to use var() references
```

## 3. Verify CSS Files Loaded

```javascript
// Run in browser DevTools console
performance.getEntriesByType('resource')
  .filter(r => r.name.includes('.css'))
  .map(r => ({
    name: r.name.split('/').pop(),
    size: r.transferSize + ' bytes'
  }));
```

**Expected result:** Should show `atom.css` (~73KB) and `tokens.css` loaded.

## 4. Check Component Classes

```javascript
// Run in browser DevTools console
const btn = document.querySelector('[data-button]');
console.log({
  classes: btn?.className,
  background: getComputedStyle(btn)?.background
});
```

**Expected result:**
- `classes`: Should be `"button button--primary button--l"` or similar
- `background`: Should be a color value, NOT `rgba(0, 0, 0, 0) none...`

## Common Issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| All tokens are `""` | Tokens not imported | Add `import '@atomchat.io/tokens/css'` to Layout |
| Tokens exist but styles don't apply | Wrong import order | Move tokens import BEFORE css import |
| `atom.css` not found in network | Package not installed | Run `pnpm add @atomchat.io/css@latest` |
| Components have classes but no styles | CSS file corrupt or blocked | Clear node_modules and reinstall |
