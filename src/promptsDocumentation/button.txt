# Component Specification

## Overview
The Button component is the primary interactive element with:
- 6 visual variants (Primary, Secondary, Tertiary, Destructive variants)
- 5 sizes (xs, s, m, l, xl)
- Loading state with spinner
- Icon support (left, right, or both)
- Disabled state
- Can render as `<button>` or `<a>` tag
- Token-driven styling (NEVER hardcode values)

## NPM Package Information

**Package:** `@atomchat.io/components-astro@2.0.2`
**Component path:** `src/atoms/Button.astro`
**Dependencies:**
- `@atomchat.io/tokens@1.0.2` (design tokens)
- `@atomchat.io/css@0.2.2` (compiled CSS from tokens)

**Optional dependencies (for animations):**
- `@atomchat.io/animations` (GSAP animation functions)
- `gsap` (animation library)

## Props API

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | "Primary" \| "Secondary" \| "Tertiary" \| "Destructive Primary" \| "Destructive Secondary" \| "Destructive Tertiary" | "Primary" | Visual style variant |
| size | "xs" \| "s" \| "m" \| "l" \| "xl" | "m" | Size of the button |
| href | string? | — | If provided, renders as `<a>` tag |
| type | "button" \| "submit" \| "reset" | "button" | Button type (only for `<button>`) |
| disabled | boolean? | false | Disables interaction |
| loading | boolean? | false | Shows loading spinner |
| class | string? | — | Additional CSS classes |

## Slots/Children

| Slot | Description |
|------|-------------|
| default | Button label text (required) |
| icon-left | Icon placed before label |
| icon-right | Icon placed after label |

## DOM Structure (Default State)

```html
<button class="button button--primary button--m" data-button data-hover-rotate>
  <span class="button__label-wrap">
    <!-- Original label -->
    <span class="button__label" data-button-label>
      <span class="button__icon" data-icon-left><!-- icon-left --></span>
      Button Text
      <span class="button__icon" data-icon-right><!-- icon-right --></span>
    </span>

    <!-- Clone label (for animation, hidden by default) -->
    <span class="button__label button__label--clone" data-button-label-clone aria-hidden="true">
      <span class="button__icon"><!-- icon-left clone --></span>
      Button Text
      <span class="button__icon"><!-- icon-right clone --></span>
    </span>
  </span>
```

## Accessibility Requirements

**ARIA attributes:**
```html
<!-- Disabled -->
<button disabled aria-disabled="true">...</button>

<!-- Loading -->
<button disabled aria-disabled="true" aria-busy="true">...</button>

<!-- Clone label -->
<span aria-hidden="true">...</span>
```

**Keyboard navigation:**
- Button must be focusable with Tab
- Activatable with Enter or Space
- Focus-visible styling required

**Motion preferences:**
- Respect `prefers-reduced-motion: reduce`
- Disable animations when OS preference is set
- Slower animation speeds for reduced motion

## Critical Rules

1. **NEVER hardcode values** — Every color, spacing, font size must use design tokens
2. **Pre-render clone labels** — Do not create clones with JavaScript (SSR compatibility)
3. **Maintain data-attribute contract** — Required for animation system to work
4. **Respect accessibility** — All ARIA attributes, keyboard navigation, focus states
5. **Follow separation of concerns** — Component renders markup, animations initialize globally
6. **Token-first philosophy** — If a token exists, use it. If it doesn't, request it.
