# Component Specification

## Overview
The Badge component is a notification indicator that displays count information with smart overflow handling. It auto-hides when count is zero and includes context-aware capping rules.

**Key Features:**
- 3 visual types (neutral, inbox, info)
- 3 visual states (default, focused, subtle)
- Smart overflow capping (default: 1-99 then "99+", inbox: 1-49 then "+50")
- Automatic hiding when count is zero
- Dynamic ARIA labels for accessibility
- Data attributes for styling hooks
- Token-driven styling (NEVER hardcode values)

## Props API

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| count | number | 0 | Notification count to display |
| context | "default" \| "inbox" | "default" | Overflow rule context |
| type | "neutral" \| "inbox" \| "info" | "neutral" | Visual type variant |

| class | string? | — | Additional CSS classes |

### Return Value
- **Rendered:** When `count > 0`
- **Hidden (null):** When `count <= 0`

## Overflow Logic

### Default Context
```
Count Input → Display Output
1-99        → Exact number (1, 5, 45, 99)
100+        → "99+"
0 or neg    → Hidden (null)
```

**Use case:** General notifications, unread counts, generic counters

### Inbox Context
```
Count Input → Display Output
1-49        → Exact number (1, 25, 49)
50+         → "+50"
0 or neg    → Hidden (null)
```

**Use case:** Email inbox, message counts, capacity-limited contexts

## Implementation Logic

```typescript
// Overflow calculation function
function formatBadgeCount(count: number, context: 'default' | 'inbox'): string {
  if (count <= 0) return ''  // Hide badge

  if (context === 'inbox') {
    return count >= 50 ? '+50' : count.toString()
  }

  // Default context
  return count > 99 ? '99+' : count.toString()
}

// ARIA label function (shows actual count, not display value)
function generateAriaLabel(count: number): string {
  if (count <= 0) return ''
  if (count === 1) return '1 notification'
  return `${count} notifications`
}
```

## Animation System (if enabled)

### Data Attributes Contract
- `data-badge` — Root element
- `data-badge-text` — Text element for animations
- `data-count` — Current count (for state tracking)
- `data-context` — Context type (default | inbox)

### Entrance Animation
```css
/* Initial state (hidden) */
.badge[data-badge] {
  transform: scale(0);
  opacity: 0;
}

/* Entrance keyframe */
@keyframes badge-entrance {
  from {
    transform: scale(0);
    opacity: 0;
  }
  to {
    transform: scale(1);
    opacity: 1;
  }
}

.badge[data-badge] {
  animation: badge-entrance 0.3s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}
```

### GSAP Animation (optional, if animations enabled)

**Entrance animation:**
```typescript
gsap.from('[data-badge]', {
  scale: 0,
  opacity: 0,
  duration: 0.3,
  ease: 'back.out(1.7)'
})
```

**Update pulse animation:**
```typescript
// When count changes
gsap.to('[data-badge-text]', {
  scale: 1.2,
  duration: 0.2,
  ease: 'power2.out',
  yoyo: true,
  repeat: 1
})
```

**Exit animation (when count becomes 0):**
```typescript
gsap.to('[data-badge]', {
  scale: 0,
  opacity: 0,
  duration: 0.2,
  ease: 'power2.in',
  onComplete: () => {
    // Component unmounts/returns null
  }
})
```

### Motion Preferences
```css

    animation-duration: 1s;
  }

    animation: none !important;
  }

  role="status"

</span>
```

**Automatic behaviors:**
- `role="status"` for live announcements
- `aria-label` reflects actual count (not display value)
- Singular/plural: "1 notification" vs "5 notifications"
- Badge hidden when count is zero (no announcements)

2. **Auto-hide when count <= 0** — Component must return null, not render empty badge
3. **Accurate ARIA labels** — aria-label shows actual count (150), even if display shows "99+"
4. **Context-aware overflow** — Default uses 99+, inbox uses +50
5. **Singular/plural logic** — "1 notification" vs "5 notifications"
6. **Token-first philosophy** — If a token exists, use it
