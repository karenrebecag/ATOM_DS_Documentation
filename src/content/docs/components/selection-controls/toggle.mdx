## Overview

The Toggle component is a switch control for representing binary on/off states. It provides visual feedback through animated knob movement and is ideal for settings, preferences, and feature toggles. It follows the ARIA switch pattern with full semantic accessibility.

**Key Features:**
- Binary state toggle (on/off) with smooth animation
- 2 size options (s, m) for compact and standard layouts
- Supporting text for additional context
- Native error state with error messages
- SVG error icon indicator
- ARIA switch role with aria-checked attribute
- Animated knob with track background
- Supports disabled state
- Conditional rendering (simple or extended field)
- Data-attribute contract for animations
- Fully token-driven styling
- Fully accessible with semantic HTML and ARIA

## Anatomy

The Toggle component has two rendering modes:

### Simple Mode (no label, text, or error)

```
┌──────────────────────┐
│  <label>             │  ← Toggle wrapper
│  ┌────────────────┐  │
│  │ <input/>       │  │  ← Hidden checkbox input
│  └────────────────┘  │
│  ┌────────────────┐  │
│  │ <span> track   │  │  ← Background track
│  │ ┌────────────┐ │  │
│  │ │ <span> knob│ │  │  ← Animated knob
│  │ └────────────┘ │  │
│  └────────────────┘  │
└──────────────────────┘
```

### Extended Mode (with label, text, or error)

```
┌──────────────────────────────┐
│  <div> .toggle-field         │  ← Field wrapper
│  ┌────────────────────────┐  │
│  │  <div> .row            │  │  ← Row container
│  │  ┌──────────────────┐  │  │
│  │  │ <div> .selector  │  │  │  ← Toggle control
│  │  │ <label> .toggle  │  │  │
│  │  │ <input/>         │  │  │
│  │  │ <span> track     │  │  │
│  │  │ <span> knob      │  │  │
│  │  └──────────────────┘  │  │
│  │  ┌──────────────────┐  │  │
│  │  │ <div> .text      │  │  │  ← Label & text
│  │  │ <label> title    │  │  │
│  │  │ <span> supportive│  │  │
│  │  └──────────────────┘  │  │
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │ <div> .error          │  │  ← Error (if present)
│  │ <svg> error-icon      │  │
│  │ <span> error-text     │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | — | Unique identifier for the toggle (required) |
| `size` | `"s" \| "m"` | `"s"` | Toggle size (compact or standard) |
| `checked` | `boolean?` | `false` | Initial on/off state |
| `disabled` | `boolean?` | `false` | Disables interaction |
| `supportiveText` | `string?` | — | Helper text below label |
| `error` | `boolean?` | `false` | Indicates validation error |
| `errorText` | `string?` | — | Error message (only shown if `error=true`) |
| `class` | `string?` | — | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Toggle label text (optional) |

**Slot Usage:**
```astro

</Toggle>
```

## Variants

### Size Variants

| Size | Track Width | Track Height | Use Case |
|------|------------|--------------|----------|
| **s** | `36px` | `20px` | Compact forms, dense layouts, mobile |
| **m** | `44px` | `24px` | Standard forms, settings, desktop |

**Size Tokens:**
```css
--toggle-size-s-width: 36px;
--toggle-size-s-height: 20px;

--toggle-size-m-width: 44px;
--toggle-size-m-height: 24px;
```

### State Variants

| State | Behavior | Visual Change |
|-------|----------|---------------|
| **Off** | Interactive | Track on left, knob at start |
| **On** | Selected | Track on right, knob at end |
| **Hover** | On mouse over | Subtle background transition |
| **Focus** | On keyboard focus | Focus ring outline |
| **Disabled** | Non-interactive | Gray track + `cursor: not-allowed` |
| **Error** | Validation failed | Red border + error message + icon |

### Rendering Modes

| Mode | Trigger | Structure |
|------|---------|-----------|
| **Simple** | No label, supportive text, or error | Single `<label>` wrapper |
| **Extended** | Has label/text/error | `<div class="toggle-field">` wrapper |

## Animations

The Toggle component integrates with the **ATOM animation system** for **knob movement** effects.

### Knob Animation

**What it does:**
When a toggle is clicked, the knob smoothly slides between on/off positions with accompanying track color transitions.

**Data attributes:**
```html
<label class="toggle" data-toggle>
  <input type="checkbox" class="toggle__input" role="switch" />
  <span class="toggle__track" data-toggle-track>
    <span class="toggle__knob" data-toggle-knob></span>
  </span>
</label>
```

**Animation Behavior:**
- **On Toggle:** Knob slides from start to end (or vice versa)
- **Duration:** 300ms (smooth transition)
- **Easing:** `ease-out` (natural deceleration)
- **Track Color:** Transitions from off to on color

**CSS-based Animation:**

```css
.toggle__knob {
  transition: transform 300ms ease-out;
}

.toggle__input:checked ~ .toggle__track {
  transition: background-color 300ms ease-out;
}

.toggle__input:checked ~ .toggle__track .toggle__knob {
  transform: translateX(var(--toggle-knob-translate-s));
}
```

**GSAP Timeline (optional enhancement):**

```typescript
// When toggle is clicked
gsap.to('[data-toggle-knob]', {
  x: isChecked ? 20 : 0,
  duration: 0.3,
  ease: 'power2.out'
});

gsap.to('[data-toggle-track]', {
  backgroundColor: isChecked ? '#22c55e' : '#e5e7eb',
  duration: 0.3,
  ease: 'power2.out'
});
```

### Initialization

**Global initialization (recommended):**

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Respects Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  .toggle__knob,
  .toggle__track {
    transition-duration: 1s;
  }
}
```

## Accessibility

### ARIA Attributes

The Toggle component includes built-in accessibility features:

```html
<!-- Off state -->
<label class="toggle" for="dark-mode">
  <input
    type="checkbox"
    id="dark-mode"
    name="dark-mode"
    role="switch"
    aria-checked="false"
    aria-label="Dark mode"
  />
  <span class="toggle__track" aria-hidden="true">
    <span class="toggle__knob"></span>
  </span>
</label>

<!-- On state -->
<label class="toggle" for="dark-mode">
  <input
    type="checkbox"
    id="dark-mode"
    name="dark-mode"
    role="switch"
    aria-checked="true"
    aria-label="Dark mode"
  />
  <span class="toggle__track" aria-hidden="true">
    <span class="toggle__knob"></span>
  </span>
</label>

<!-- Toggle with error -->
<div class="toggle-field toggle-field--error">
  <div class="toggle-field__row">
    <label class="toggle" for="required">
      <input
        type="checkbox"
        id="required"
        name="required"
        role="switch"
        aria-checked="false"
        aria-invalid="true"
        aria-describedby="error-msg"
      />
      <span class="toggle__track" aria-hidden="true">
        <span class="toggle__knob"></span>
      </span>
    </label>
  </div>
  <div class="toggle-field__error" id="error-msg" role="alert">
    <svg aria-hidden="true"><!-- Error icon --></svg>
    <span>This feature must be enabled</span>
  </div>
</div>
```

**Automatic behaviors:**
- `role="switch"` for proper switch semantics
- `aria-checked="true|false"` reflects toggle state
- Track and knob hidden with `aria-hidden="true"`
- Error messages with `role="alert"`
- `aria-invalid="true"` for validation errors
- `aria-describedby` links to error messages

### Best Practices

**Provide descriptive labels:**
```astro
<Toggle name="dark-mode" size="m">
  Enable dark mode
</Toggle>
```

**Use supporting text for clarity:**
```astro
<Toggle
  name="analytics"
  size="m"
  supportiveText="Help improve our service by sharing usage data"
>
  Share analytics
</Toggle>
```

**Show error messages clearly:**
```astro
<Toggle
  name="terms"
  size="m"
  error
  errorText="You must agree to continue"
>
  I agree to terms
</Toggle>
```

**Avoid unclear labels:**
```astro
<!-- Bad: Too vague -->
<Toggle name="feature" size="m">
  Enable
</Toggle>

<!-- Good: Clear action -->
<Toggle name="notifications" size="m">
  Email notifications
</Toggle>
```

**Don't toggle on hover:**
```astro
<!-- Bad: Unexpected behavior -->
<Toggle name="hidden" size="m">Hidden by default</Toggle>

<!-- Good: Click to toggle -->
<Toggle name="visible" size="m">Show advanced options</Toggle>
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Toggle component is controlled by design tokens:

- **Global consistency** — All toggles use the same size and color scale
- **Easy theming** — Switch themes by updating token values
- **State management** — Off/on/hover/focus/error states via tokens
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Custom toggle variant for important settings */
.toggle--important {
  --toggle-bg-checked: var(--primitive-error-500);
  --toggle-border-checked: var(--primitive-error-600);
  --toggle-field-bg-error: var(--primitive-error-50);
}

.toggle--important:hover {
  --toggle-bg-hovered: var(--primitive-error-600);
}
```

**Usage:**
```astro
<Toggle name="critical-setting" class="toggle--important" size="m">
  Enable critical feature
</Toggle>
```
