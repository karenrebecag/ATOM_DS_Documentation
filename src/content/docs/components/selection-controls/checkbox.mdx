## Overview

The Checkbox component is a binary selection control that allows users to choose one or multiple options from a set. It supports checked, unchecked, and disabled states with smooth animations and full keyboard accessibility.

**Key Features:**
- Binary selection (checked/unchecked)
- Support for multiple checkboxes in groups
- Radio button variant via `type` prop
- Light and dark theme support
- SVG checkmark with smooth animations
- Custom styling via design tokens
- Fully accessible with semantic HTML and ARIA labels
- Data-attribute contract for animations
- Respects `prefers-reduced-motion`

## Anatomy

The Checkbox component consists of a semantic label wrapping an input element with a custom styled indicator:

```
┌─────────────────────────────────┐
│  <label> .checkbox              │  ← Wrapper label
│  ┌──────────────┐               │
│  │ <input/>     │               │  ← Hidden native input
│  │ .checkbox__  │               │
│  │ input        │               │
│  └──────────────┘               │
│  ┌──────────────┐               │
│  │ <span>       │               │  ← Custom indicator
│  │ .checkbox__  │               │
│  │ custom       │               │
│  │ ┌──────────┐ │               │
│  │ │ <svg>    │ │               │  ← SVG checkmark
│  │ │ checkmark│ │               │
│  │ └──────────┘ │               │
│  └──────────────┘               │
│  ┌──────────────┐               │
│  │ <span>       │               │  ← Label text
│  │ .checkbox__  │               │
│  │ label        │               │
│  └──────────────┘               │
└─────────────────────────────────┘
```

**DOM Structure:**
```html
<label class="checkbox checkbox--light" for="agree-terms" data-checkbox>
  <input
    class="checkbox__input"
    type="checkbox"
    id="agree-terms"
    name="terms"
    value="agree"
  />
  <span class="checkbox__custom" aria-hidden="true" data-checkbox-custom>
    <svg
      class="checkbox__check"
      viewBox="0 0 12 10"
      fill="none"
      data-checkbox-check
    >
      <path
        d="M1 5.5L4 8.5L11 1.5"
        stroke="currentColor"
        stroke-width="1.5"
        stroke-linecap="round"
        stroke-linejoin="round"
      />
    </svg>
  </span>
  <span class="checkbox__label">
    I agree to the terms
  </span>
</label>
```

### Props

| Prop | Type | Default | Description |

| `name` | `string` | — | Form input name attribute (required) |

| `theme` | `"light" \| "dark"` | `"light"` | Visual theme variant |

## Variants

### Theme Variants

| Theme | Background | Check Color | Use Case |
|-------|------------|-------------|----------|
| **light** | `--checkbox-bg-light` | `--checkbox-fg-light` | Light backgrounds, default theme |
| **dark** | `--checkbox-bg-dark` | `--checkbox-fg-dark` | Dark backgrounds, high contrast |

### Input Type Variants

| Type | Behavior | Shape | Use Case |
|------|----------|-------|----------|
| **checkbox** | Multiple selection | Square with rounded corners | Multiple options allowed |
| **radio** | Single selection | Circle | Only one option per group |

### State Variants

| State | Behavior | Visual Change |
|-------|----------|---------------|
| **Unchecked** | Interactive, clickable | Empty indicator |
| **Checked** | Shows checkmark | Filled indicator + SVG checkmark |
| **Hover** | On mouse over | Background color transitions |
| **Focus** | On keyboard focus | Focus ring outline |
| **Active** | On click/press | Checkmark animation (scale/fade) |
| **Disabled** | Non-interactive | Gray background + `cursor: not-allowed` |

## Animations

The Checkbox component integrates with the **ATOM animation system** (`@atomchat.io/animations`) for **checkmark animation** effects.

### Checkmark Animation

**What it does:**
When a checkbox is toggled, the checkmark SVG animates with scale and fade effects for visual feedback.

**Data attributes:**
```html
<label class="checkbox" data-checkbox>
  <input type="checkbox" class="checkbox__input" />
  <span class="checkbox__custom" data-checkbox-custom>
    <svg class="checkbox__check" data-checkbox-check>
      <path />
    </svg>
  </span>
  <span class="checkbox__label">Label</span>
</label>
```

**Animation Triggers:**
- **On Check:** Checkmark scales from 0 to 1 and fades in
- **On Uncheck:** Checkmark scales from 1 to 0 and fades out
- **Duration:** 200ms (default)
- **Easing:** `ease-out` (smooth deceleration)

**GSAP Timeline Implementation:**

```typescript
// When checkbox is checked
gsap.to('[data-checkbox-check]', {
  scale: 1,
  opacity: 1,
  duration: 0.2,
  ease: 'atom'
});

// When checkbox is unchecked
gsap.to('[data-checkbox-check]', {
  scale: 0,
  opacity: 0,
  duration: 0.2,
  ease: 'atom'
});
```

### Customization

```css
/* Adjust animation duration */
.checkbox {
  --animation-duration: 300ms;
}

/* Customize scale effect */
.checkbox__check {
  transform-origin: center;
  --scale-from: 0.8;
  --scale-to: 1;
}
```

### Initialization

**Global initialization (recommended):**

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Respects Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  [data-checkbox-check] {
    animation-duration: 1s; /* Slower animation */
  }
}
```

The Checkbox component includes built-in accessibility features:

```html

    id="subscribe"

    <svg class="checkbox__check" aria-hidden="true">

    </svg>
  </span>
  <span class="checkbox__label">Subscribe</span>

<!-- Disabled checkbox -->
<label class="checkbox checkbox--disabled" for="legacy">
  <input
    type="checkbox"
    id="legacy"
    name="legacy"
    disabled
    aria-disabled="true"
  />
  <span class="checkbox__custom" aria-hidden="true">
    <svg class="checkbox__check" aria-hidden="true">
      <path />
    </svg>
  </span>
  <span class="checkbox__label">Disabled option</span>

<!-- Checkbox group -->
<fieldset>
  <legend>Select options</legend>
  <div role="group" aria-labelledby="options-legend">
    <Checkbox name="option" value="1">Option 1</Checkbox>
    <Checkbox name="option" value="2">Option 2</Checkbox>
  </div>

```

**Automatic behaviors:**
- Native `<input type="checkbox">` for semantic HTML
- `aria-checked` reflects checked state
- SVG indicator hidden with `aria-hidden="true"`
- Label click toggles checkbox
- Keyboard: Space/Enter to toggle, Tab to navigate

### Best Practices

**Use fieldset for groups:**
```astro
<fieldset>
  <legend>Email Preferences</legend>
  <Checkbox name="emails" value="marketing">
    Marketing emails
  </Checkbox>
  <Checkbox name="emails" value="updates">
    Product updates
  </Checkbox>
</fieldset>
```

**Provide descriptive labels:**
```astro
<Checkbox name="terms" value="accept">
  I agree to the Terms of Service and Privacy Policy
</Checkbox>
```

**Use radio buttons for single selection:**
```astro
<Checkbox type="radio" name="size" value="small">
  Small
</Checkbox>
<Checkbox type="radio" name="size" value="large">
  Large
</Checkbox>
```

**Associate labels with inputs:**
```astro
<Checkbox name="newsletter" value="yes">
  Subscribe to newsletter
</Checkbox>
```

**Avoid disabling without context:**
```astro
<!-- Bad: No explanation -->
<Checkbox name="option" value="yes" disabled>
  Unavailable option
</Checkbox>

<!-- Good: Provides reason -->
<div>
  <Checkbox name="option" value="yes" disabled>
    Premium feature (upgrade required)
  </Checkbox>
</div>
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Checkbox component is controlled by design tokens. This ensures:

- **Global consistency** — All checkboxes use the same size and color scale
- **Easy theming** — Switch themes by changing token values
- **State management** — Hover/focus/active states via tokens
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Custom checkbox variant for error state */
.checkbox--error {
  --checkbox-border-enabled: var(--primitive-error-500);
  --checkbox-border-focused: var(--primitive-error-600);
  --checkbox-bg-checked: var(--primitive-error-500);
  --checkbox-fg-checked: var(--primitive-neutral-0);
}

.checkbox--error:hover {
  --checkbox-bg-enabled: var(--primitive-error-50);
}
```

**Usage:**
```astro
<Checkbox name="confirm" value="yes" class="checkbox--error">
  I understand the consequences
</Checkbox>
```
