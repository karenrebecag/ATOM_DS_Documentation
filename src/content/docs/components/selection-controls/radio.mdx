## Overview

The Radio component is a single-selection control that allows users to choose exactly one option from a mutually exclusive set. It extends beyond basic radio input with built-in support for descriptive labels, supporting text, size variants, and error messaging.

**Key Features:**
- Single selection within a group (mutually exclusive)
- 2 size options (s, m) for compact and standard layouts
- Supporting text for additional context
- Native error state with error messages
- SVG error icon indicator
- Supports custom label and helper content
- Responsive to parent container sizing
- Full keyboard accessibility
- Conditional rendering (simple or extended field)
- Fully token-driven styling
- Accessible with semantic HTML and ARIA labels

## Anatomy

The Radio component has two rendering modes:

### Simple Mode (no label, text, or error)

```
┌──────────────────┐
│  <label>         │  ← Radio wrapper
│  ┌────────────┐  │
│  │ <input/>   │  │  ← Hidden native input
│  └────────────┘  │
│  ┌────────────┐  │
│  │ <span>     │  │  ← Custom circle indicator
│  │ .radio__   │  │
│  │ circle     │  │
│  │ ┌────────┐ │  │
│  │ │ <span> │ │  │  ← Animated dot
│  │ │ .radio │ │  │
│  │ │ __dot  │ │  │
│  │ └────────┘ │  │
│  └────────────┘  │
└──────────────────┘
```

### Extended Mode (with label, text, or error)

```
┌──────────────────────────────┐
│  <div> .radio-field          │  ← Field wrapper
│  ┌────────────────────────┐  │
│  │  <div> .radio-field__  │  │  ← Row container
│  │  row                   │  │
│  │  ┌──────────────────┐  │  │
│  │  │ <div> .selector  │  │  │  ← Radio control
│  │  │ <label> .radio   │  │  │
│  │  │ <input/>         │  │  │
│  │  │ <span> circle    │  │  │
│  │  │ <span> dot       │  │  │
│  │  └──────────────────┘  │  │
│  │  ┌──────────────────┐  │  │
│  │  │ <div> .text      │  │  │  ← Label & text
│  │  │ <label> title    │  │  │
│  │  │ <span> supportive│  │  │
│  │  └──────────────────┘  │  │
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │ <div> .error          │  │  ← Error (if present)
│  │ <svg> error-icon      │  │
│  │ <span> error-text     │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | — | Radio group name (required) |
| `value` | `string?` | — | Option value (required for groups) |
| `size` | `"s" \| "m"` | `"s"` | Indicator size (compact or standard) |
| `checked` | `boolean?` | `false` | Initial selected state |
| `disabled` | `boolean?` | `false` | Disables interaction |
| `supportiveText` | `string?` | — | Helper text below label |
| `error` | `boolean?` | `false` | Indicates validation error |
| `errorText` | `string?` | — | Error message (only shown if `error=true`) |
| `class` | `string?` | — | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Radio label text |

**Slot Usage:**
```astro
<Radio name="choice" value="option1" size="m">
  Label text for this option
</Radio>
```

## Variants

### Size Variants

| Size | Circle Diameter | Use Case |
|------|-----------------|----------|
| **s** | `16px` | Compact forms, dense layouts, mobile |
| **m** | `20px` | Standard forms, desktop, default UI |

**Size Token:**
```css
--radio-size-s: 16px;
--radio-size-m: 20px;
```

### State Variants

| State | Behavior | Visual Change |
|-------|----------|---------------|
| **Unchecked** | Interactive | Empty circle indicator |
| **Checked** | Selected in group | Animated dot appears |
| **Hover** | On mouse over | Subtle background highlight |
| **Focus** | On keyboard focus | Focus ring outline |
| **Disabled** | Non-interactive | Gray circle + `cursor: not-allowed` |
| **Error** | Validation failed | Red border + error message + icon |
| **Success** | Valid selection | Green checkmark (optional) |

### Rendering Modes

| Mode | Trigger | Structure |
|------|---------|-----------|
| **Simple** | No label, supportive text, or error | Single `<label>` wrapper |
| **Extended** | Has label/text/error | `<div class="radio-field">` wrapper |

## Animations

The Radio component integrates with the **ATOM animation system** for **dot animation** effects.

### Dot Animation

**What it does:**
When a radio button is selected, the dot animates into the circle with scale and fade effects.

**Data attributes:**
```html
<label class="radio" data-radio>
  <input type="radio" class="radio__input" />
  <span class="radio__circle" data-radio-circle>
    <span class="radio__dot" data-radio-dot></span>
  </span>
</label>
```

**Animation Behavior:**
- **On Check:** Dot scales from 0 to 1 and fades in (200ms)
- **On Uncheck:** Dot scales from 1 to 0 and fades out (200ms)
- **Easing:** `ease-out` (smooth deceleration)

**GSAP Timeline:**

```typescript
// When radio is selected
gsap.to('[data-radio-dot]', {
  scale: 1,
  opacity: 1,
  duration: 0.2,
  ease: 'atom'
});

// When radio is deselected
gsap.to('[data-radio-dot]', {
  scale: 0,
  opacity: 0,
  duration: 0.2,
  ease: 'atom'
});
```

### Initialization

**Global initialization (recommended):**

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Respects Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  [data-radio-dot] {
    animation-duration: 1s;
  }
}
```

## Accessibility

### ARIA Attributes

The Radio component includes built-in accessibility features:

```html
<!-- Unchecked radio -->
<label class="radio" for="option-1" data-radio>
  <input
    type="radio"
    id="option-1"
    name="choice"
    value="option-1"
    aria-label="Option 1"
  />
  <span class="radio__circle" aria-hidden="true">
    <span class="radio__dot"></span>
  </span>
</label>

<!-- Checked radio -->
<label class="radio" for="option-2" data-radio>
  <input
    type="radio"
    id="option-2"
    name="choice"
    value="option-2"
    checked
    aria-label="Option 2 (selected)"
  />
  <span class="radio__circle" aria-hidden="true">
    <span class="radio__dot"></span>
  </span>
</label>

<!-- Radio with error -->
<div class="radio-field radio-field--error">
  <div class="radio-field__row">
    <label class="radio" for="invalid" data-radio>
      <input
        type="radio"
        id="invalid"
        name="choice"
        value="invalid"
        aria-invalid="true"
        aria-describedby="error-msg"
      />
      <span class="radio__circle" aria-hidden="true">
        <span class="radio__dot"></span>
      </span>
    </label>
  </div>
  <div class="radio-field__error" id="error-msg" role="alert">
    <svg aria-hidden="true"><!-- Error icon --></svg>
    <span>This option is unavailable</span>
  </div>
</div>

<!-- Radio group -->
<fieldset>
  <legend>Select your preference</legend>
  <Radio name="pref" value="1">Option 1</Radio>
  <Radio name="pref" value="2">Option 2</Radio>
</fieldset>
```

**Automatic behaviors:**
- Native `<input type="radio">` for semantic HTML
- `<fieldset>` and `<legend>` for grouping
- Radio buttons with same `name` are automatically grouped
- Error icon hidden with `aria-hidden="true"`
- Error messages with `role="alert"`
- `aria-invalid="true"` for validation errors

### Best Practices

**Use fieldset and legend:**
```astro
<fieldset>
  <legend>Shipping Method</legend>
  <Radio name="shipping" value="standard">Standard</Radio>
  <Radio name="shipping" value="express">Express</Radio>
</fieldset>
```

**Provide clear labels:**
```astro
<Radio name="agree" value="yes">
  I agree to the Terms of Service
</Radio>
```

**Use supporting text for clarity:**
```astro
<Radio
  name="option"
  value="premium"
  supportiveText="Includes priority support and updates"
>
  Premium Plan
</Radio>
```

**Show error messages clearly:**
```astro
<Radio
  name="choice"
  value="unavailable"
  error
  errorText="This option is temporarily unavailable"
>
  Unavailable Option
</Radio>
```

**Don't rely on visual cues alone:**
```astro
<!-- Bad: No label text -->
<Radio name="agree" value="yes" />

<!-- Good: Descriptive label -->
<Radio name="agree" value="yes">
  I agree to proceed
</Radio>
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Radio component is controlled by design tokens:

- **Global consistency** — All radios use the same size and color scale
- **Easy theming** — Switch themes by updating token values
- **State management** — Hover/focus/active/error states via tokens
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Custom radio variant for premium selection */
.radio--premium {
```
