## Overview

The Stack component is a foundational layout primitive that arranges its children vertically with consistent spacing between them. It's built on CSS Flexbox and provides a simple, declarative API for vertical layouts throughout your application.

:::tip[Interactive Playground]
**[Open Stack Playground →](/playground/stack)**

View all Stack configurations with live spacing adjustments and alignment options in an isolated environment.
:::

**Key Features:**
- Vertical flex layout (column direction)
- 5 gap sizes (xs, s, m, l, xl) powered by design tokens
- Flexible horizontal alignment options (start, center, end, stretch)
- Vertical distribution control (start, center, end, between, around)
- Full-height mode for viewport-spanning layouts
- Polymorphic element rendering (div, section, nav, etc.)
- Zero hardcoded values — fully token-driven
- Responsive by default

## Anatomy

The Stack component is a simple container that uses CSS Flexbox to arrange children vertically:

```
┌──────────────────────────┐
│  .stack (container)      │  ← Flexbox column
│  ┌────────────────────┐  │
│  │  Child 1           │  │
│  └────────────────────┘  │
│         ↓ gap            │
│  ┌────────────────────┐  │
│  │  Child 2           │  │
│  └────────────────────┘  │
│         ↓ gap            │
│  ┌────────────────────┐  │
│  │  Child 3           │  │
│  └────────────────────┘  │
└──────────────────────────┘
```

**DOM Structure:**
```html
<div class="stack stack--gap-m stack--align-stretch">
  <div>Child 1</div>
  <div>Child 2</div>
  <div>Child 3</div>
</div>
```

**With custom alignment:**
```html
<section class="stack stack--gap-l stack--align-center stack--justify-between stack--full-height">
  <header>Header</header>
  <main>Content</main>
  <footer>Footer</footer>
</section>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `gap` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Space between children (uses design tokens) |
| `align` | `"start" \| "center" \| "end" \| "stretch"` | `"stretch"` | Horizontal alignment (align-items in Flexbox) |
| `justify` | `"start" \| "center" \| "end" \| "between" \| "around"` | `"start"` | Vertical distribution (justify-content in Flexbox) |
| `fullHeight` | `boolean?` | `false` | Makes stack take full viewport height (`100vh`) |
| `as` | `string?` | `"div"` | HTML element to render as (div, section, nav, aside, etc.) |
| `class` | `string?` | — | Additional CSS classes |

## Variants

### Gap Sizes

Stack uses the design system's spacing scale for consistent gap values across all components:

| Gap | Token | Value | Use Case |
|-----|-------|-------|----------|
| **xs** | `--gap-xs` | 8px | Tight spacing (form labels, list items) |
| **s** | `--gap-s` | 12px | Compact spacing (card content, navigation) |
| **m** | `--gap-m` | 16px | Default spacing (paragraphs, sections) |
| **l** | `--gap-l` | 24px | Generous spacing (major sections, cards) |
| **xl** | `--gap-xl` | 32px | Extra spacing (page sections, hero blocks) |

### Alignment Options

#### Horizontal Alignment (`align`)

| Value | Behavior | Visual |
|-------|----------|--------|
| **start** | Align children to the left edge | `├─┤ ` |
| **center** | Align children to horizontal center | ` ┼ ` |
| **end** | Align children to the right edge | ` ├─┤` |
| **stretch** | Stretch children to full width (default) | `├───┤` |

#### Vertical Distribution (`justify`)

| Value | Behavior | Visual |
|-------|----------|--------|
| **start** | Pack children at the top (default) | `▄▄▄  ` |
| **center** | Pack children in the middle | ` ▄▄▄ ` |
| **end** | Pack children at the bottom | `  ▄▄▄` |
| **between** | Distribute with space between | `▄ ▄ ▄` |
| **around** | Distribute with space around | ` ▄ ▄ ▄ ` |

## Structure Tokens

Stack uses design tokens for all spacing values, ensuring consistency across the design system:

### Gap Tokens (from `@atomchat.io/tokens`)

```css
--gap-xs: 8px;
--gap-s: 12px;
--gap-m: 16px;   /* Default gap */
--gap-l: 24px;
--gap-xl: 32px;
```

### Component Classes

```css
/* Base */
.stack {
  display: flex;
  flex-direction: column;
}

/* Gap modifiers */
.stack--gap-xs { gap: var(--gap-xs); }
.stack--gap-s  { gap: var(--gap-s); }
.stack--gap-m  { gap: var(--gap-m); }  /* Default */
.stack--gap-l  { gap: var(--gap-l); }
.stack--gap-xl { gap: var(--gap-xl); }

/* Alignment modifiers */
.stack--align-start   { align-items: flex-start; }
.stack--align-center  { align-items: center; }
.stack--align-end     { align-items: flex-end; }
.stack--align-stretch { align-items: stretch; }  /* Default */

/* Justify modifiers */
.stack--justify-start   { justify-content: flex-start; }  /* Default */
.stack--justify-center  { justify-content: center; }
.stack--justify-end     { justify-content: flex-end; }
.stack--justify-between { justify-content: space-between; }
.stack--justify-around  { justify-content: space-around; }

/* Full-height modifier */
.stack--full-height {
  min-height: 100vh;
}
```

## Accessibility

### Semantic HTML

Stack supports rendering as any HTML element via the `as` prop:

```astro
<!-- Navigation sidebar -->
<Stack as="nav" gap="m">...</Stack>

<!-- Article section -->
<Stack as="article" gap="l">...</Stack>

<!-- Form layout -->
<Stack as="form" gap="m">...</Stack>

<!-- Aside content -->
<Stack as="aside" gap="s">...</Stack>
```

**Best Practices:**
- ✅ Use semantic elements that match the content's purpose
- ✅ Use `<section>` for thematic grouping
- ✅ Use `<nav>` for navigation menus
- ✅ Use `<form>` for form layouts
- ❌ Don't use `<div>` when a semantic element is available

### Reading Order

Stack maintains natural document flow, which is crucial for screen readers:

```astro
<!-- ✅ GOOD: Logical reading order -->
<Stack gap="m">
  <h2>Article Title</h2>
  <p class="meta">By Jane Doe • March 31, 2026</p>
  <p>Article content...</p>
  <a href="#read-more">Read More</a>
</Stack>

<!-- ❌ BAD: Visual-only ordering (use justify instead) -->
<Stack gap="m" class="reversed">
  <a href="#read-more">Read More</a>
  <p>Article content...</p>
  <p class="meta">By Jane Doe • March 31, 2026</p>
  <h2>Article Title</h2>
</Stack>
```

### Focus Management

When using Stack for interactive lists, ensure proper focus order:

```astro
<Stack gap="s" as="nav" role="navigation" aria-label="Main navigation">
  <a href="#home">Home</a>
  <a href="#about">About</a>
  <a href="#contact">Contact</a>
</Stack>
```

## Design Tokens in Action

Stack is a perfect example of the token-first philosophy. Every spacing value comes from the design system:

### Token Override Example

You can override gap tokens for specific use cases:

```astro
<Stack gap="m" style="--gap-m: 20px;">
  <!-- Custom medium gap for this instance -->
  <p>Paragraph 1</p>
  <p>Paragraph 2</p>
</Stack>
```

### Custom Gap with Design Tokens

```astro
<Stack gap="m" class="custom-stack">
  <div>Item 1</div>
  <div>Item 2</div>
</Stack>

<style>
  .custom-stack {
    /* Use a different token for this stack */
    --gap-m: var(--gap-l);
  }
</style>
```
