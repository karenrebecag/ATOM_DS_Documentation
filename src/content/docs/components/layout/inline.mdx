## Overview

The Inline component is a foundational layout primitive that arranges its children horizontally with consistent spacing between them. It's built on CSS Flexbox and wraps content by default for responsive behavior, making it perfect for tag lists, button groups, breadcrumbs, and horizontal navigation.

:::tip[Interactive Playground]
**[Open Inline Playground →](/playground/inline)**

View all Inline configurations with live spacing adjustments and wrapping behavior in an isolated environment.
:::

**Key Features:**
- Horizontal flex layout (row direction)
- Wraps content by default for responsive layouts
- 5 gap sizes (xs, s, m, l, xl) powered by design tokens
- Flexible vertical alignment options (start, center, end, baseline)
- Horizontal distribution control (start, center, end, between, around)
- Optional nowrap mode for strict horizontal layouts
- Polymorphic element rendering (div, nav, ul, etc.)
- Zero hardcoded values — fully token-driven

## Anatomy

The Inline component uses CSS Flexbox to arrange children horizontally with wrapping:

```
┌────────────────────────────────────────┐
│  .inline (container)                   │  ← Flexbox row
│  ┌──────┐ gap ┌──────┐ gap ┌──────┐  │
│  │Child1│ ↔   │Child2│ ↔   │Child3│  │
│  └──────┘     └──────┘     └──────┘  │
│  ↓ (wraps if needed)                  │
│  ┌──────┐ gap ┌──────┐                │
│  │Child4│ ↔   │Child5│                │
│  └──────┘     └──────┘                │
└────────────────────────────────────────┘
```

**DOM Structure:**
```html
<div class="inline inline--gap-m inline--align-center">
  <span>Tag 1</span>
  <span>Tag 2</span>
  <span>Tag 3</span>
</div>
```

**With custom alignment:**
```html
<nav class="inline inline--gap-s inline--align-baseline inline--justify-between">
  <a href="#home">Home</a>
  <a href="#about">About</a>
  <a href="#contact">Contact</a>
</nav>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `gap` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Space between children (uses design tokens) |
| `align` | `"start" \| "center" \| "end" \| "baseline"` | `"center"` | Vertical alignment (align-items in Flexbox) |
| `justify` | `"start" \| "center" \| "end" \| "between" \| "around"` | `"start"` | Horizontal distribution (justify-content in Flexbox) |
| `nowrap` | `boolean?` | `false` | Prevents content from wrapping to next line |
| `as` | `string?` | `"div"` | HTML element to render as (div, nav, ul, etc.) |
| `class` | `string?` | — | Additional CSS classes |

## Variants

### Gap Sizes

Inline uses the same spacing scale as Stack for consistency:

| Gap | Token | Value | Use Case |
|-----|-------|-------|----------|
| **xs** | `--gap-xs` | 8px | Tight spacing (icon + text, breadcrumbs) |
| **s** | `--gap-s` | 12px | Compact spacing (tags, chips) |
| **m** | `--gap-m` | 16px | Default spacing (button groups, nav links) |
| **l** | `--gap-l` | 24px | Generous spacing (hero buttons, features) |
| **xl** | `--gap-xl` | 32px | Extra spacing (section elements) |

### Alignment Options

#### Vertical Alignment (`align`)

| Value | Behavior | Visual | Use Case |
|-------|----------|--------|----------|
| **start** | Align to top | `▀──` | Icons with text |
| **center** | Align to middle (default) | `─▄─` | Tags, badges, buttons |
| **end** | Align to bottom | `──▄` | Footer elements |
| **baseline** | Align to text baseline | `A─a` | Mixed typography |

#### Horizontal Distribution (`justify`)

| Value | Behavior | Visual | Use Case |
|-------|----------|--------|----------|
| **start** | Pack items left (default) | `▄▄▄  ` | Tag lists |
| **center** | Pack items center | ` ▄▄▄ ` | Centered navigation |
| **end** | Pack items right | `  ▄▄▄` | Right-aligned actions |
| **between** | Distribute with space between | `▄ ▄ ▄` | Header layouts |
| **around** | Distribute with space around | ` ▄ ▄ ▄ ` | Social icons |

### Wrapping Behavior

| Mode | Behavior | Use Case |
|------|----------|----------|
| **Default (wraps)** | Content wraps to next line when width is exceeded | Tag lists, responsive navigation |
| **nowrap={true}** | Content stays in single line, may overflow | Strict horizontal layouts, scrollable areas |

## Structure Tokens

Inline uses the same spacing tokens as Stack:

### Gap Tokens (from `@atomchat.io/tokens`)

```css
--gap-xs: 8px;
--gap-s: 12px;
--gap-m: 16px;   /* Default gap */
--gap-l: 24px;
--gap-xl: 32px;
```

### Component Classes

```css
/* Base */
.inline {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;  /* Wraps by default */
}

/* Gap modifiers */
.inline--gap-xs { gap: var(--gap-xs); }
.inline--gap-s  { gap: var(--gap-s); }
.inline--gap-m  { gap: var(--gap-m); }  /* Default */
.inline--gap-l  { gap: var(--gap-l); }
.inline--gap-xl { gap: var(--gap-xl); }

/* Alignment modifiers */
.inline--align-start    { align-items: flex-start; }
.inline--align-center   { align-items: center; }  /* Default */
.inline--align-end      { align-items: flex-end; }
.inline--align-baseline { align-items: baseline; }

/* Justify modifiers */
.inline--justify-start   { justify-content: flex-start; }  /* Default */
.inline--justify-center  { justify-content: center; }
.inline--justify-end     { justify-content: flex-end; }
.inline--justify-between { justify-content: space-between; }
.inline--justify-around  { justify-content: space-around; }

/* Nowrap modifier */
.inline--nowrap {
  flex-wrap: nowrap;
}
```

## Accessibility

### Semantic HTML

Use appropriate HTML elements via the `as` prop:

```astro
<!-- Navigation -->
<Inline as="nav" gap="m" aria-label="Main navigation">...</Inline>

<!-- List of items -->
<Inline as="ul" gap="s" role="list">...</Inline>

<!-- Breadcrumbs -->
<Inline as="nav" gap="xs" aria-label="Breadcrumb">...</Inline>
```

**Best Practices:**
- ✅ Use `<nav>` for navigation menus
- ✅ Use `<ul>` for lists of similar items
- ✅ Add `aria-label` for navigation contexts
- ✅ Use `aria-current="page"` for active breadcrumb items
- ❌ Don't use Inline for stacking content vertically (use Stack instead)

### Focus Order

Inline maintains natural tab order:

```astro
<!-- ✅ GOOD: Left-to-right focus order -->
<Inline gap="m" as="nav">
  <a href="#home">Home</a>       <!-- Tab 1 -->
  <a href="#about">About</a>     <!-- Tab 2 -->
  <a href="#contact">Contact</a> <!-- Tab 3 -->
</Inline>
```

### Wrapping Behavior

When content wraps, ensure visual hierarchy remains clear:

```astro
<!-- Tags with clear visual grouping -->
<section>
  <h3>Skills</h3>
  <Inline gap="s">
    <Tag>JavaScript</Tag>
    <Tag>TypeScript</Tag>
    <Tag>React</Tag>
    <!-- Wraps naturally on small screens -->
  </Inline>
</section>
```
