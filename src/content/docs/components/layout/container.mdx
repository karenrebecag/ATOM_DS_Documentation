## Overview

The Container component is a fundamental layout primitive that constrains content to a maximum width and centers it horizontally. It provides six size variants based on common breakpoints, ensuring content remains readable and well-proportioned across all screen sizes.

:::tip[Interactive Playground]
**[Open Container Playground →](/playground/container)**

Explore all Container sizes with live width adjustments and nested content examples in an isolated environment.
:::

**Key Features:**
- Six size variants (sm, md, lg, xl, 2xl, full)
- Automatic horizontal centering with padding
- Responsive padding that adapts to viewport width
- Max-width constraints for optimal readability
- Polymorphic element rendering
- Zero hardcoded values — fully token-driven
- Mobile-first responsive design
- Works seamlessly with other layout primitives

## Anatomy

The Container component creates a centered, max-width wrapper with responsive padding:

```
┌─────────────────────────────────────────┐
│  Viewport (full width)                  │
│  ╔═════════════════════════════════╗    │
│  ║  .container (centered)          ║    │  ← Auto margins center
│  ║  padding: var(--container-px)   ║    │  ← Responsive padding
│  ║  max-width: var(--container-*)  ║    │  ← Constrained width
│  ║  ┌───────────────────────────┐  ║    │
│  ║  │  Content (full width)     │  ║    │
│  ║  └───────────────────────────┘  ║    │
│  ╚═════════════════════════════════╝    │
└─────────────────────────────────────────┘
```

**DOM Structure:**
```html
<div class="container container--lg">
  <h1>Centered Content</h1>
  <p>Content is constrained to max-width and centered horizontally.</p>
</div>
```

**Nested containers:**
```html
<div class="container container--xl">
  <header>Full width header within container</header>
  <div class="container container--md">
    <article>Narrower content for better readability</article>
  </div>
</div>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"sm" \| "md" \| "lg" \| "xl" \| "2xl" \| "full"` | `"lg"` | Maximum width of the container |
| `as` | `string?` | `"div"` | HTML element to render as (div, section, main, article, etc.) |
| `class` | `string?` | — | Additional CSS classes |

## Variants

### Size Options

Container provides six size variants optimized for different content types:

| Size | Max-Width | Use Case |
|------|-----------|----------|
| **sm** | 640px | Blog posts, narrow text content, single-column forms |
| **md** | 768px | Documentation, articles, reading-optimized content |
| **lg** | 1024px | General page layouts, default content width (default) |
| **xl** | 1280px | Dashboards, wide tables, multi-column layouts |
| **2xl** | 1536px | Full-featured applications, complex dashboards |
| **full** | none | Edge-to-edge layouts with padding only |

### Responsive Padding

Container uses responsive horizontal padding that adapts to viewport width:

| Viewport Width | Padding | Token |
|----------------|---------|-------|
| **< 640px** | 16px | `--gap-m` |
| **640px - 1024px** | 24px | `--gap-l` |
| **> 1024px** | 32px | `--gap-xl` |

This ensures content never touches the viewport edges while maximizing usable space.

## Structure Tokens

Container uses design tokens for max-width constraints and responsive padding:

### Width Tokens (from `@atomchat.io/tokens`)

```css
--container-sm: 640px;
--container-md: 768px;
--container-lg: 1024px;
--container-xl: 1280px;
--container-2xl: 1536px;

/* Responsive padding */
--container-px: var(--gap-m);  /* 16px on mobile */

@media (min-width: 640px) {
  --container-px: var(--gap-l);  /* 24px on tablet */
}

@media (min-width: 1024px) {
  --container-px: var(--gap-xl);  /* 32px on desktop */
}
```

### Component Classes

```css
.container {
  width: 100%;
  margin-left: auto;
  margin-right: auto;
  padding-left: var(--container-px);
  padding-right: var(--container-px);
}

.container--sm   { max-width: var(--container-sm); }
.container--md   { max-width: var(--container-md); }
.container--lg   { max-width: var(--container-lg); }
.container--xl   { max-width: var(--container-xl); }
.container--2xl  { max-width: var(--container-2xl); }
.container--full { max-width: none; }
```

## Accessibility

### Semantic HTML

Container supports rendering as any HTML element via the `as` prop:

```astro
<!-- Main content wrapper -->
<Container as="main">...</Container>

<!-- Article wrapper -->
<Container as="article">...</Container>

<!-- Section wrapper -->
<Container as="section">...</Container>

<!-- Header wrapper -->
<Container as="header">...</Container>
```

**Best Practices:**
- ✅ Use `<main>` for primary page content
- ✅ Use `<article>` for self-contained content (blog posts, news articles)
- ✅ Use `<section>` for thematic grouping
- ✅ Use `<header>` or `<footer>` for site-wide headers/footers
- ❌ Don't nest landmark elements incorrectly (e.g., `<main>` inside `<main>`)

### Landmark Navigation

Container with semantic elements creates navigable landmarks for screen readers:

```astro
<Container as="header">
  <nav aria-label="Main navigation">
    <!-- Navigation links -->
  </nav>
</Container>

<Container as="main">
  <h1>Main Content</h1>
  <!-- Primary content -->
</Container>

<Container as="footer">
  <p>© 2026 Company Name</p>
</Container>
```

### Focus Management

Container doesn't interfere with focus order, maintaining natural tab flow:

```astro
<Container>
  <a href="#content">Skip to content</a>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
    <a href="/contact">Contact</a>
  </nav>
</Container>
```
