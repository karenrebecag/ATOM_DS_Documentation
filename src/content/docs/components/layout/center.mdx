## Overview

The Center component is a utility primitive that centers its children both horizontally and vertically using CSS Flexbox. It's perfect for centering single elements, creating splash screens, loading states, and vertically centering content within containers.

**Key Features:**
- Horizontal and vertical centering with Flexbox
- Row or column direction
- Optional gap spacing (xs, s, m, l, xl)
- Full-height mode for viewport-height centering
- Full-width mode for edge-to-edge layouts
- Inline mode for inline-flex behavior
- Polymorphic element rendering
- Zero hardcoded values — fully token-driven
- Perfect for loading states, empty states, and splash screens

## Anatomy

The Center component uses Flexbox to center content in both dimensions:

```
┌─────────────────────────────────────────┐
│  .center (container)                    │
│                                         │
│          ┌──────────────┐               │
│          │   Centered   │               │  ← justify-content: center
│          │   Content    │               │  ← align-items: center
│          └──────────────┘               │
│                                         │
└─────────────────────────────────────────┘
```

**DOM Structure (basic):**
```html
<div class="center center--row">
  <p>Centered content</p>
</div>
```

**With gap spacing:**
```html
<div class="center center--column center--gap-m">
  <img src="/icon.svg" alt="Icon" />
  <h2>Title</h2>
  <p>Description</p>
</div>
```

**Full-height centering:**
```html
<div class="center center--row center--full-height">
  <div class="loading-spinner">Loading...</div>
</div>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `direction` | `"row" \| "column"` | `"row"` | Flex direction (row = horizontal, column = vertical) |
| `gap` | `"xs" \| "s" \| "m" \| "l" \| "xl"?` | — | Space between children (optional) |
| `fullHeight` | `boolean?` | `false` | Makes container take full viewport height (`100vh`) |
| `fullWidth` | `boolean?` | `false` | Makes container take full width (`100%`) |
| `inline` | `boolean?` | `false` | Uses `inline-flex` instead of `flex` |
| `as` | `string?` | `"div"` | HTML element to render as (div, section, main, etc.) |
| `class` | `string?` | — | Additional CSS classes |

---

## Variants

### Direction Modes

| Direction | Behavior | Visual | Use Case |
|-----------|----------|--------|----------|
| **row** | Horizontal centering (default) | `→ ● ←` | Centering single elements, buttons, icons |
| **column** | Vertical stack with centering | `↑ ● ↓` | Splash screens, empty states, loading screens |

### Gap Sizes (Optional)

Center supports optional gap spacing when multiple children are present:

| Gap | Token | Value | Use Case |
|-----|-------|-------|----------|
| **xs** | `--gap-xs` | 8px | Tight spacing (icon + label) |
| **s** | `--gap-s` | 12px | Compact spacing (small groups) |
| **m** | `--gap-m` | 16px | Default spacing (modal content) |
| **l** | `--gap-l` | 24px | Generous spacing (splash screens) |
| **xl** | `--gap-xl` | 32px | Extra spacing (hero sections) |

### Full-Height Mode

When `fullHeight={true}`, the Center component takes full viewport height:

```astro
<Center fullHeight>
  <div class="loading-spinner">Loading...</div>
</Center>
```

Perfect for:
- Loading screens
- Splash pages
- 404 pages
- Empty states
- Modal overlays

## Accessibility

### Semantic HTML

Center supports rendering as any HTML element via the `as` prop:

```astro
<!-- Main content wrapper -->
<Center as="main" fullHeight>...</Center>

<!-- Section wrapper -->
<Center as="section" direction="column" gap="l">...</Center>

<!-- Article wrapper -->
<Center as="article">...</Center>
```

**Best Practices:**
- ✅ Use `<main>` for primary centered content
- ✅ Use `<section>` for thematic centered sections
- ✅ Add `aria-label` or `aria-labelledby` when semantic meaning isn't clear
- ✅ Include loading announcements with `aria-live` for loading states
- ❌ Don't overuse Center for all layouts (use semantically appropriate components)

### Loading States

Provide proper loading announcements for screen readers:

```astro
<Center fullHeight>
  <div role="status" aria-live="polite" aria-label="Loading">
    <div class="spinner"></div>
    <span class="sr-only">Loading content, please wait...</span>
  </div>
</Center>

<style>
  .sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
  }
</style>
```

### Focus Management

Ensure proper focus for centered interactive content:

```astro
<Center direction="column" gap="m" fullHeight>
  <h1>Welcome</h1>
  <p>Get started with your account</p>
  <button autofocus>Continue</button>
</Center>
```
