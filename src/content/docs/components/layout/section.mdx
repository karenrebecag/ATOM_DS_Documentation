## Overview

The Section component is a semantic layout primitive that provides consistent vertical padding for page sections. It's designed to create visual hierarchy and rhythm throughout your pages with five size variants, ensuring predictable spacing between major content blocks.

:::tip[Interactive Playground]
**[Open Section Playground →](/playground/section)**

Explore all Section padding sizes and nesting patterns with live adjustments in an isolated environment.
:::

**Key Features:**
- Semantic `<section>` element by default
- Five padding sizes (xs, s, m, l, xl) powered by design tokens
- Vertical padding only (preserves horizontal flow)
- Polymorphic element rendering
- Perfect for page layout rhythm and visual hierarchy
- Zero hardcoded values — fully token-driven
- Responsive padding that adapts to viewport width
- Composable with Container and other layout primitives

## Anatomy

The Section component adds consistent vertical padding to create breathing room between page sections:

```
┌─────────────────────────────────────────┐
│  .section (semantic <section>)          │
│  ┌─────────────────────────────────┐    │
│  │  padding-top: var(--section-*)  │    │  ← Vertical padding
│  │  ┌───────────────────────────┐  │    │
│  │  │  Content (full width)     │  │    │  ← No horizontal constraints
│  │  └───────────────────────────┘  │    │
│  │  padding-bottom: var(--sect-*) │    │  ← Vertical padding
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

**DOM Structure:**
```html
<section class="section section--m">
  <h2>Section Title</h2>
  <p>Section content with medium vertical padding.</p>
</section>
```

**Composed with Container:**
```html
<section class="section section--l">
  <div class="container container--lg">
    <h2>Centered, Constrained Content</h2>
    <p>Section provides vertical spacing, Container provides width constraint.</p>
  </div>
</section>
```

## Variants

### Padding Sizes

Section provides five padding sizes for creating vertical rhythm:

| Size | Padding (Y-axis) | Token | Use Case |
|------|------------------|-------|----------|
| **xs** | 24px | `--section-xs` | Tight sections, footer, compact areas |
| **s** | 48px | `--section-s` | Sub-sections, card groups, compact layouts |
| **m** | 80px | `--section-m` | Standard sections, default spacing (default) |
| **l** | 120px | `--section-l` | Major sections, hero areas, emphasis |
| **xl** | 160px | `--section-xl` | Extra emphasis, splash sections, spacious layouts |

### Responsive Padding

Section padding adapts to viewport width for better mobile experience:

| Viewport Width | Multiplier | Example (m) |
|----------------|------------|-------------|
| **< 640px** | 0.6× | 48px (mobile) |
| **640px - 1024px** | 0.8× | 64px (tablet) |
| **> 1024px** | 1× | 80px (desktop) |

This ensures sections don't consume excessive vertical space on small screens.

### Responsive Adjustments

```css
/* Mobile (< 640px) */
@media (max-width: 639px) {
  --section-xs: 16px;   /* 24px × 0.6 */
  --section-s: 32px;    /* 48px × 0.6 */
  --section-m: 48px;    /* 80px × 0.6 */
  --section-l: 72px;    /* 120px × 0.6 */
  --section-xl: 96px;   /* 160px × 0.6 */

/* Tablet (640px - 1024px) */
@media (min-width: 640px) and (max-width: 1023px) {
  --section-xs: 20px;   /* 24px × 0.8 */

  --section-m: 64px;    /* 80px × 0.8 */
```

## Accessibility

### Semantic HTML

Section renders as `<section>` by default, providing semantic meaning to page structure:

```astro
<!-- ✅ GOOD: Semantic section structure -->
<Section>
  <h2>Features</h2>
  <p>Feature content...</p>
</Section>

<!-- ✅ GOOD: Article content -->
<Section as="article">
  <h1>Blog Post Title</h1>
  <p>Post content...</p>
</Section>

<!-- ✅ GOOD: Main content area -->
<Section as="main">
  <h1>Page Title</h1>
  <p>Main content...</p>
</Section>
```

**Best Practices:**
- ✅ Use `<section>` for thematic grouping of content
- ✅ Each section should have a heading (h1-h6)
- ✅ Use `as="article"` for self-contained content
- ✅ Use `as="main"` for primary page content
- ✅ Add `aria-label` or `aria-labelledby` when heading isn't immediately available
- ❌ Don't use Section just for styling without semantic meaning

### ARIA Labels

Provide labels for sections without visible headings:

```astro
<Section aria-label="Product features">
  <Grid columns={3} gap="l">
    <div>Feature 1</div>
    <div>Feature 2</div>
    <div>Feature 3</div>
  </Grid>
</Section>
```

### Heading Hierarchy

Maintain proper heading order within sections:

```astro
<!-- ✅ GOOD: Proper heading hierarchy -->
<Section>
  <h2>Main Topic</h2>
  <Section size="s">
    <h3>Subtopic</h3>
    <p>Content...</p>
  </Section>
</Section>

<!-- ❌ BAD: Skipped heading level -->
<Section>
  <h2>Main Topic</h2>
  <Section size="s">
    <h4>Subtopic</h4> <!-- Skipped h3 -->
    <p>Content...</p>
  </Section>
</Section>
```
