## Overview

The Divider component is a semantic `<hr>` element that creates visual separation between content sections. It's lightweight, accessible, and supports custom opacity for flexible styling.

### Key Features

- **Semantic HTML**: Proper `<hr>` element for accessibility
- **Accessibility**: Marked aria-hidden="true" (decorative)
- **Custom Opacity**: Adjustable transparency for subtle to prominent dividers
- **Token-Driven**: Uses design tokens for consistent styling
- **Lightweight**: Minimal markup, single-element component
- **Flexible**: Works in any layout context

## Anatomy

The Divider component is a simple semantic `<hr>` element:

```html
<hr class="divider" data-divider aria-hidden="true" />

<!-- With custom opacity -->
<hr
  class="divider"
  style="--divider-opacity: 0.5"
  data-divider
  aria-hidden="true"
/>
```

**DOM Structure:**
- `<hr>` - Semantic horizontal rule element
- `.divider` - Root component class
- `data-divider` - Component identifier
- `aria-hidden="true"` - Marks as decorative element
- `style="--divider-opacity: [value]"` - Applied for custom opacity (0-1)

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `opacity` | `number` | `undefined` | Custom opacity value (0-1, where 0 is transparent and 1 is opaque) |
| `class` | `string` | `""` | Additional CSS classes |

### Data Attributes

| Attribute | Values | Purpose |
|-----------|--------|---------|
| `data-divider` | Present (no value) | Component identifier |
| `aria-hidden` | "true" | Marks divider as decorative |

## Structure Tokens

The Divider component uses these structural tokens:

| Token | Purpose |
|-------|---------|
| `--divider-height` | Divider line thickness |
| `--divider-color` | Default divider color |
| `--divider-opacity` | Opacity level (customizable) |

## Dependencies

The Divider component has no component dependencies. It uses design tokens for styling:

**Design Tokens Required:**
- `--divider-height` - Line thickness
- `--divider-color` - Line color
- `--divider-opacity` - Opacity value

## Accessibility

### Semantic HTML

- **Uses `<hr>` element**: Proper semantic meaning as a thematic break
- **aria-hidden="true"**: Marks divider as decorative (not content)
- **Screen reader impact**: Dividers are hidden from screen readers (appropriate for visual separators)

### Best Practices

- Use dividers for visual organization, not as semantic content separators
- Don't use multiple dividers in quick succession (confuses screen reader users)
- Combine with semantic sections (use `<section>` or `<article>`) for proper document structure
- Consider whether content sections should be announced to screen readers

### Screen Reader Testing

```html
<!-- Good: Divider between semantic sections -->
<section>Section 1 content</section>
<hr aria-hidden="true" />
<section>Section 2 content</section>

<!-- Good: Divider in visually organized lists -->
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>
<hr aria-hidden="true" />
<ul>
  <li>Item 3</li>
  <li>Item 4</li>
</ul>

<!-- Avoid: Using dividers as primary content structure -->
<!-- Instead: Use semantic HTML sections -->
```
