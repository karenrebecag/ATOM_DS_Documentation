## Overview

The Grid component is a powerful layout primitive that uses CSS Grid to create responsive multi-column layouts. It provides a declarative API for grid-based designs with automatic responsiveness and consistent spacing powered by design tokens.

:::tip[Interactive Playground]
**[Open Grid Playground →](/playground/grid)**

Explore all Grid configurations with live column adjustments, gap sizes, and alignment options in an isolated environment.
:::

**Key Features:**
- CSS Grid-based multi-column layouts
- Fixed columns (1-6) or auto-fit responsive modes
- 5 gap sizes (xs, s, m, l, xl) powered by design tokens
- Item alignment control (start, center, end, stretch)
- Content justification options
- Preserve mode to prevent auto-spanning
- Polymorphic element rendering
- Zero hardcoded values — fully token-driven
- Mobile-first responsive design

## Anatomy

The Grid component creates a CSS Grid container with configurable columns and spacing:

```
┌─────────────────────────────────────────┐
│  .grid (container)                      │  ← CSS Grid
│  ┌──────────┐  ┌──────────┐  ┌────────┐│
│  │  Cell 1  │  │  Cell 2  │  │ Cell 3 ││  ← Grid items
│  └──────────┘  └──────────┘  └────────┘│
│        ↕ gap         ↕ gap       ↕      │
│  ┌──────────┐  ┌──────────┐  ┌────────┐│
│  │  Cell 4  │  │  Cell 5  │  │ Cell 6 ││
│  └──────────┘  └──────────┘  └────────┘│
└─────────────────────────────────────────┘
```

**DOM Structure (3-column grid):**
```html
<div class="grid grid--cols-3 grid--gap-m">
  <div>Cell 1</div>
  <div>Cell 2</div>
  <div>Cell 3</div>
  <div>Cell 4</div>
  <div>Cell 5</div>
  <div>Cell 6</div>
</div>
```

**Auto-fit responsive grid:**
```html
<div class="grid grid--auto-fit grid--gap-l" data-grid>
  <article>Card 1</article>
  <article>Card 2</article>
  <article>Card 3</article>
</div>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `columns` | `1 \| 2 \| 3 \| 4 \| 5 \| 6 \| "auto-fit" \| "auto-fit-sm" \| "auto-fit-lg"` | `3` | Number of columns or auto-fit mode |
| `gap` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Space between grid items (uses design tokens) |
| `align` | `"start" \| "center" \| "end" \| "stretch"` | `"stretch"` | Vertical alignment of grid items (align-items) |
| `justify` | `"start" \| "center" \| "end" \| "stretch"` | `"stretch"` | Horizontal alignment of grid items (justify-items) |
| `preserve` | `boolean?` | `false` | Prevents automatic spanning to fill gaps |
| `as` | `string?` | `"div"` | HTML element to render as (div, section, ul, etc.) |
| `class` | `string?` | — | Additional CSS classes |

## Variants

### Column Modes

Grid supports fixed columns (1-6) and three auto-fit responsive modes:

| Mode | Behavior | Min Column Width | Use Case |
|------|----------|------------------|----------|
| **1-6** | Fixed number of columns at all breakpoints | Auto-calculated | Precise control, custom responsive |
| **auto-fit** | Auto-fills with 250px minimum columns | 250px | General purpose cards/items |
| **auto-fit-sm** | Auto-fills with smaller 200px columns | 200px | Compact cards, avatars, chips |
| **auto-fit-lg** | Auto-fills with larger 300px columns | 300px | Product cards, feature blocks |

### Gap Sizes

Grid uses the design system's spacing scale for consistent gap values:

| Gap | Token | Value | Use Case |
|-----|-------|-------|----------|
| **xs** | `--gap-xs` | 8px | Tight spacing (form grids, compact lists) |
| **s** | `--gap-s` | 12px | Compact spacing (card grids, galleries) |
| **m** | `--gap-m` | 16px | Default spacing (general layouts) |
| **l** | `--gap-l` | 24px | Generous spacing (feature grids, sections) |
| **xl** | `--gap-xl` | 32px | Extra spacing (hero grids, large cards) |

### Alignment Options

#### Item Alignment (`align`)

| Value | Behavior | Visual |
|-------|----------|--------|
| **start** | Align items to top of cell | `▄   ` |
| **center** | Align items to center of cell | ` ▄  ` |
| **end** | Align items to bottom of cell | `  ▄ ` |
| **stretch** | Stretch items to fill cell height (default) | `▀▀▀▀` |

#### Item Justification (`justify`)

| Value | Behavior | Visual |
|-------|----------|--------|
| **start** | Justify items to left of cell | `├─┤ ` |
| **center** | Justify items to center of cell | ` ┼ ` |
| **end** | Justify items to right of cell | ` ├─┤` |
| **stretch** | Stretch items to fill cell width (default) | `├───┤` |

## Structure Tokens

Grid uses design tokens for all spacing values, ensuring consistency across the design system:

### Gap Tokens (from `@atomchat.io/tokens`)

```css
--gap-xs: 8px;
--gap-s: 12px;
--gap-m: 16px;
--gap-l: 24px;
--gap-xl: 32px;
```

### Component Classes

```css
.grid {
  display: grid;
}

/* Fixed columns */
.grid--cols-1 { grid-template-columns: repeat(1, 1fr); }
.grid--cols-2 { grid-template-columns: repeat(2, 1fr); }
.grid--cols-3 { grid-template-columns: repeat(3, 1fr); }
.grid--cols-4 { grid-template-columns: repeat(4, 1fr); }
.grid--cols-5 { grid-template-columns: repeat(5, 1fr); }
.grid--cols-6 { grid-template-columns: repeat(6, 1fr); }

/* Auto-fit modes */
.grid--auto-fit    { grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); }
.grid--auto-fit-sm { grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); }
.grid--auto-fit-lg { grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); }

/* Gap variants */
.grid--gap-xs { gap: var(--gap-xs); }
.grid--gap-s  { gap: var(--gap-s); }
.grid--gap-m  { gap: var(--gap-m); }
.grid--gap-l  { gap: var(--gap-l); }
.grid--gap-xl { gap: var(--gap-xl); }

/* Alignment */
.grid--align-start   { align-items: start; }
.grid--align-center  { align-items: center; }
.grid--align-end     { align-items: end; }

.grid--justify-start   { justify-items: start; }
.grid--justify-center  { justify-items: center; }
.grid--justify-end     { justify-items: end; }

/* Preserve mode */
.grid--preserve {
  grid-auto-flow: dense;
}
```

## Accessibility

### Semantic HTML

Grid supports rendering as any HTML element via the `as` prop:

```astro
<!-- Section grid -->
<Grid as="section" columns={3} gap="l">...</Grid>

<!-- List grid -->
<Grid as="ul" columns="auto-fit" gap="m">
  <li>Item 1</li>
  <li>Item 2</li>
</Grid>

<!-- Navigation grid -->
<Grid as="nav" columns={4} gap="s">...</Grid>
```

**Best Practices:**
- ✅ Use `<section>` for thematic content groups
- ✅ Use `<ul>` or `<ol>` for lists of items
- ✅ Use `<nav>` for navigation menus
- ✅ Add ARIA labels when semantic meaning isn't clear
- ❌ Don't use Grid purely for visual layout if content has semantic meaning

### Reading Order

Grid maintains natural document flow by default:

```astro
<!-- ✅ GOOD: Logical reading order matches visual order -->
<Grid columns={2} gap="m">
  <article>Post 1</article>
  <article>Post 2</article>
  <article>Post 3</article>
  <article>Post 4</article>
</Grid>

<!-- ❌ BAD: CSS reordering breaks screen reader flow -->
<Grid columns={2} gap="m">
  <article style="order: 4">Post 4</article>
  <article style="order: 1">Post 1</article>
  <article style="order: 3">Post 3</article>
  <article style="order: 2">Post 2</article>
</Grid>
```

### Focus Management

Ensure proper focus order in interactive grids:

```astro
<Grid columns={3} gap="m" role="group" aria-label="Product grid">
  <article tabindex="0">
    <h3>Product 1</h3>
    <button>Add to Cart</button>
  </article>
  <article tabindex="0">
    <h3>Product 2</h3>
    <button>Add to Cart</button>
  </article>
  <article tabindex="0">
    <h3>Product 3</h3>
```
