## Overview

The TextField component is a single-line text input that supports labels, leading/trailing icon slots, supportive text, character counters, and four semantic states: default, filled, error, and disabled. Sizes range from xs to xl.

**Key Features:**
- 5 sizes (xs, s, m, l, xl)
- Leading and trailing icon/action slots
- Supportive text with optional icon
- Character counter
- States: default · hover · focus · filled · error · disabled
- Fully token-driven — no hardcoded values
- Accessible label association

## Anatomy

```
┌─────────────────────────────┐
│ .text-field                 │
│  .text-field__label         │  ← Field label
│  .text-field__field         │  ← Input container (border, bg)
│  │  .text-field__leading    │  ← Optional icon / prefix
│  │  .text-field__input      │  ← Native <input>
│  │  .text-field__trailing   │  ← Optional icon / action
│  .text-field__bottom        │
│  │  .text-field__supportive │  ← Helper text + icon
│  │  .text-field__counter    │  ← Character counter (optional)
└─────────────────────────────┘
```

**DOM Structure:**
```html
<div class="text-field text-field--m">
  <label class="text-field__label">Email</label>
  <div class="text-field__field">
    <span class="text-field__leading"><!-- icon --></span>
    <input class="text-field__input" type="email" placeholder="you@example.com" />
    <span class="text-field__trailing"><!-- action --></span>
  </div>
  <div class="text-field__bottom">
    <span class="text-field__supportive">
      <span class="text-field__supportive-icon"><!-- icon --></span>
      <span class="text-field__supportive-text">Enter your work email</span>
    </span>
    <span class="text-field__counter">0 / 100</span>
  </div>
</div>
```

## API Reference

### Modifier Classes

| Class | Description |
|-------|-------------|
| `text-field--xs/s/m/l/xl` | Size variant |
| `text-field--filled` | Applied when input has a value |
| `text-field--error` | Error state — red border + label |
| `text-field--disabled` | Disabled state |

### States

| State | Trigger | Visual |
|-------|---------|--------|
| **Default** | Initial | Gray border |
| **Hover** | `:hover` on field | Darker border |
| **Focus** | `:focus` on input | Primary border + shadow |
| **Filled** | `.text-field--filled` | Filled border color |
| **Error** | `.text-field--error` | Red border, red label |
| **Disabled** | `.text-field--disabled` | Muted bg + text |

### Size Variants

| Size | Use Case |
|------|---------|
| `xs` | Compact tables, dense UIs |
| `s` | Toolbars, secondary forms |
| `m` | Default — most forms |
| `l` | Prominent single inputs |
| `xl` | Hero inputs, search bars |

## Usage

### Basic Input

```html
<div class="text-field text-field--m">
  <label class="text-field__label">Full Name</label>
  <div class="text-field__field">
    <input class="text-field__input" type="text" placeholder="John Doe" />
  </div>
</div>
```

### With Supportive Text

```html
<div class="text-field text-field--m">
  <label class="text-field__label">Password</label>
  <div class="text-field__field">
    <input class="text-field__input" type="password" />
  </div>
  <div class="text-field__bottom">
    <span class="text-field__supportive">
      <span class="text-field__supportive-text">Minimum 8 characters</span>
    </span>
  </div>
</div>
```

### Error State

```html
<div class="text-field text-field--m text-field--error text-field--filled">
  <label class="text-field__label">Email</label>
  <div class="text-field__field">
    <input class="text-field__input" type="email" value="invalid" />
  </div>
  <div class="text-field__bottom">
    <span class="text-field__supportive">
      <span class="text-field__supportive-text">Please enter a valid email address</span>
    </span>
  </div>
</div>
```

### Disabled State

```html
<div class="text-field text-field--m text-field--disabled">
  <label class="text-field__label">Username</label>
  <div class="text-field__field">
    <input class="text-field__input" disabled value="johndoe" />
  </div>
</div>
```

## Accessibility

- `<label>` must be associated to `<input>` via `for`/`id` or wrapping
- Error state requires `aria-describedby` pointing to the supportive text
- Disabled inputs automatically receive `:disabled` styles
- Supportive icon is decorative — use `aria-hidden="true"`

```html
<div class="text-field text-field--m text-field--error">
  <label class="text-field__label" for="email">Email</label>
  <div class="text-field__field">
    <input
      class="text-field__input"
      id="email"
      type="email"
      aria-describedby="email-error"
      aria-invalid="true"
    />
  </div>
  <div class="text-field__bottom">
    <span class="text-field__supportive" id="email-error" role="alert">
      <span class="text-field__supportive-text">Invalid email address</span>
    </span>
  </div>
</div>
```
