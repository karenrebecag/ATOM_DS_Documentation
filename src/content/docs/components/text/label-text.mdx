## Overview

The LabelText component renders semantic HTML `<label>` elements for form inputs. It supports required indicators, multiple sizes, font weights, and states with proper accessibility.

### Key Features

- **Semantic HTML**: Proper `<label>` element for form association
- **Input Association**: `for` attribute automatically links to input elements
- **Required Indicator**: Optional asterisk (*) for required fields
- **3 Size Options**: Label, label-small, and footnote sizes
- **3 Weight Options**: Regular, medium, and bold font weights
- **Disabled State**: Visual indication for disabled form fields
- **Custom Colors**: Support for CSS color values or inherit
- **Accessibility**: Proper aria-hidden for decorative asterisk
- **Lightweight**: Simple, single-element component

## Installation

To use the LabelText component in your project, install it from the Atom Design System package:

```bash
npm install @atomchat.io/components-astro
# or
pnpm add @atomchat.io/components-astro
```

Then import and include the CSS tokens:

```astro

import '@atomchat.io/components-astro/dist/tokens.css'
---
```

## Anatomy

The LabelText component renders a semantic `<label>` element with optional required indicator:

```html
<label class="label-text label-text--label label-text--medium" for="email" data-text>
  Email address
  <span class="label-text__required" aria-hidden="true"> *</span>
</label>
```

**DOM Structure:**
- `<label>` - Semantic form label element
- `.label-text` - Root component class
- `.label-text--{size}` - Size modifier (label, label-small, footnote)
- `.label-text--{weight}` - Font weight modifier (regular, medium, bold)
- `.label-text--disabled` - Applied when disabled prop is true
- `.label-text__required` - Required indicator asterisk (aria-hidden="true")
- `for` attribute - Associated input ID for proper form linking
- `data-text` - Component identifier
- `style="color: [value]"` - Applied for custom colors

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `for` | `string` | `undefined` | ID of associated input element (recommended) |
| `size` | `"label" \| "label-small" \| "footnote"` | `"label"` | Text size for different contexts |
| `weight` | `"regular" \| "medium" \| "bold"` | `"medium"` | Font weight emphasis |
| `required` | `boolean` | `false` | Show required asterisk indicator |
| `disabled` | `boolean` | `false` | Disable label appearance |
| `color` | `string` | `"inherit"` | Text color (CSS value or inherit) |
| `class` | `string` | `""` | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Label text content (required) |

### Data Attributes

| Attribute | Values | Purpose |
|-----------|--------|---------|
| `data-text` | Present (no value) | Component identifier |

## Accessibility

### Semantic HTML

- **Always use `for` attribute** - Links label to input for proper form association
- **Required indicator** - Asterisk is marked aria-hidden="true" (decorative)
- **Paired with inputs** - Labels should always accompany form inputs

### Screen Reader Testing

```html
<!-- Good: Proper label-input association -->
<label for="email">Email Address</label>
<input type="email" id="email" />

<!-- Good: Required is announced by input, asterisk is decorative -->
<label for="password">Password <span aria-hidden="true">*</span></label>
<input type="password" id="password" required />

<!-- Avoid: Label without for attribute -->
<label>Email Address</label>
<input type="email" />

<!-- Avoid: Asterisk without aria-hidden -->
<label for="field">Field *</label>
<input type="text" id="field" />
```

### Best Practices

- Always associate labels with inputs using the `for` attribute
- Mark decorative elements (like asterisks) with aria-hidden="true"
- Don't use placeholder text as substitute for labels
- Ensure labels are visible and descriptive
- Use required prop for actual required fields
- Test with screen readers to verify proper announcement

## Composition Patterns

### Form Field Component

```astro
---
import { LabelText } from '@atomchat.io/components-astro'

interface FormFieldProps {
  id: string
  label: string
  type?: string
  required?: boolean
  disabled?: boolean
  placeholder?: string
  helpText?: string
  error?: string
}

const { id, label, type = 'text', required, disabled, placeholder, helpText, error } = Astro.props
---

<div class="form-field">
  <LabelText for={id} required={required} disabled={disabled}>
    {label}
  </LabelText>
  <input
    type={type}
    id={id}
    placeholder={placeholder}
    disabled={disabled}
    required={required}
    aria-invalid={error ? 'true' : 'false'}
    aria-describedby={error ? `${id}-error` : helpText ? `${id}-help` : undefined}
  />
  {error && (
    <span id={`${id}-error`} class="form-error">{error}</span>
  )}
  {helpText && (
    <span id={`${id}-help`} class="form-help">{helpText}</span>
  )}
</div>

<style>
  .form-field {
    display: flex;
    flex-direction: column;
    gap: var(--spacing-xs);
    margin-bottom: var(--spacing-md);
  }

  input {
    padding: var(--spacing-sm) var(--spacing-md);
    border: 1px solid var(--stroke-default);
    border-radius: var(--radius-sm);
    font-size: var(--font-size-md);
  }

  input[aria-invalid="true"] {
    border-color: var(--fg-accent-danger);
  }

  .form-error {
    font-size: var(--font-size-sm);
    color: var(--fg-accent-danger);
  }

  .form-help {
    font-size: var(--font-size-sm);
    color: var(--fg-secondary);
  }
</style>
```
