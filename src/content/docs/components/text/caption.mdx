## Overview

The Caption component is a flexible text element for auxiliary, secondary, or metadata content. It supports multiple sizes, weights, colors, and semantic HTML element types (span, p, figcaption) for proper markup and accessibility.

### Key Features

- **Flexible Element Types**: Render as `<span>`, `<p>`, or `<figcaption>` for semantic HTML
- **3 Size Options**: Caption, label, and footnote sizes for different contexts
- **2 Weight Options**: Regular and medium font weights
- **4 Color Options**: Primary, secondary, tertiary semantic colors + custom CSS support
- **3 Alignment Options**: Left, center, and right text alignment
- **Custom Color Support**: Accept any CSS color value for advanced use cases
- **Muted by Default**: Secondary color emphasis for auxiliary content
- **Lightweight**: Simple, single-element component

## Installation

To use the Caption component in your project, install it from the Atom Design System package:

```bash
npm install @atomchat.io/components-astro
# or
pnpm add @atomchat.io/components-astro
```

```astro

import '@atomchat.io/components-astro/dist/tokens.css'
---
```

## Anatomy

The Caption component renders a flexible HTML element with typography modifiers:

```html
<span class="caption caption--caption caption--regular caption--left caption--secondary" data-text>
  Caption text content
</span>

<!-- Or with semantic HTML -->
<figcaption class="caption caption--caption caption--regular caption--secondary" data-text>
  Figure caption
</figcaption>
```

**DOM Structure:**
- `.caption` - Root element (can be span, p, or figcaption)
- `.caption--{size}` - Size modifier (caption, label, footnote)
- `.caption--{weight}` - Font weight modifier (regular, medium)
- `.caption--{color}` - Semantic color modifier (primary, secondary, tertiary)
- `.caption--{align}` - Text alignment modifier (left, center, right)
- `data-text` - Component identifier
- `style="color: [value]"` - Applied only for custom CSS colors

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"caption" \| "label" \| "footnote"` | `"caption"` | Text size for different contexts |
| `weight` | `"regular" \| "medium"` | `"regular"` | Font weight emphasis |
| `color` | `"primary" \| "secondary" \| "tertiary" \| string` | `"secondary"` | Semantic color or custom CSS value |
| `align` | `"left" \| "center" \| "right"` | `"left"` | Text alignment |
| `as` | `"span" \| "p" \| "figcaption"` | `"span"` | HTML element type for semantic markup |
| `class` | `string` | `""` | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Caption text content (required) |

### Data Attributes

| Attribute | Values | Purpose |
|-----------|--------|---------|
| `data-text` | Present (no value) | Component identifier |

## Accessibility

### Semantic HTML

- **Use `as="figcaption"`** for image and figure captions
- **Use `as="p"`** for paragraph-level content blocks
- **Use `as="span"`** for inline text

### Best Practices

- Always pair captions with their primary content (images, figures, forms)
- Use semantic colors consistently for meaning
- Ensure adequate contrast between caption and background colors
- Don't rely solely on reduced opacity for critical information
- Use footnotes for supplementary, not essential information
- Test with screen readers for proper context

### Screen Reader Testing

```html
<!-- Good: Semantic figcaption -->
<figure>
  <img src="diagram.jpg" alt="System architecture" />
  <figcaption class="caption">Figure 1: Architecture overview</figcaption>
</figure>

<!-- Good: Paragraph context -->
<p>Main content here</p>
<Caption as="p">Supporting metadata</Caption>

<!-- Avoid: Span for important context -->
<Caption as="span">Critical information should not be secondary text</Caption>
```
