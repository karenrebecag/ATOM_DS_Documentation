## Overview

The Heading component renders semantic HTML headings (h1-h6) with flexible typography styling and optional GSAP animations. It decouples HTML heading level from visual size, allowing proper semantic markup while maintaining design flexibility.

### Key Features

- **Level/Size Decoupling**: Use h3 semantically styled as h1 size
- **10 Visual Sizes**: Display-xl, display-l, display-m, display-s, huge-title, h1–h4, and heading
- **Smart Auto-Mapping**: Automatic size from heading level (h1→h1 size, h2→h2 size, etc.)
- **3 Font Weights**: Regular, medium, and bold options
- **6 Letter Spacing Options**: Tighter to widest for fine-tuned typography
- **Text Balancing**: Optional text-wrap: balance for improved readability
- **7 GSAP Animations**: Masked text (lines/words/chars), scramble, fade, highlight effects
- **Flexible Triggers**: Scroll or load animations with configurable delay
- **Custom Colors**: Support for CSS color values or inherit
- **3 Alignment Options**: Left, center, right text alignment

## Installation

To use the Heading component in your project, install it from the Atom Design System package:

```bash
npm install @atomchat.io/components-astro
# or
pnpm add @atomchat.io/components-astro
```

Then import and include the CSS tokens and GSAP library (for animations):

```astro
---
import { Heading } from '@atomchat.io/components-astro'
import '@atomchat.io/components-astro/dist/tokens.css'

// Optional: For animations
import gsap from 'gsap'
import SplitText from 'gsap/SplitText'
gsap.registerPlugin(SplitText)
---
```

## Anatomy

The Heading component renders a semantic HTML heading element with typography modifiers:

```html
<h2
  class="heading heading--h2 heading--bold heading--tight heading--left"
  data-text
  data-text-animate="masked-lines"
  data-text-trigger="scroll"
  data-text-split="lines"
  data-text-delay="0"
>
  Heading text content
</h2>
```

**DOM Structure:**
- `h1`–`h6` - Semantic heading element (determined by `level` prop)
- `.heading` - Root component class
- `.heading--{size}` - Visual size modifier (display-xl, h1, h2, etc.)
- `.heading--{weight}` - Font weight modifier (regular, medium, bold)
- `.heading--{tracking}` - Letter spacing modifier (tighter, tight, normal, wide, wider, widest)
- `.heading--{align}` - Text alignment modifier (left, center, right)
- `.heading--balance` - Applied when balance prop is true (text-wrap: balance)
- `data-text` - Component identifier
- `data-text-animate` - Animation type (if animate prop provided)
- `data-text-trigger` - Animation trigger mode (scroll, load)
- `data-text-split` - SplitText split type (chars, words, lines)
- `data-text-delay` - Animation delay in seconds
- `style="color: [value]"` - Applied for custom colors

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `level` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | `2` | HTML heading level (h1–h6) |
| `size` | `"display-xl" \| "display-l" \| "display-m" \| "display-s" \| "huge-title" \| "h1" \| "h2" \| "h3" \| "h4" \| "heading"` | Auto-mapped from level | Visual size independent of level |
| `weight` | `"regular" \| "medium" \| "bold"` | `"bold"` | Font weight emphasis |
| `tracking` | `"tighter" \| "tight" \| "normal" \| "wide" \| "wider" \| "widest"` | `"tight"` | Letter spacing |
| `color` | `string` | `"inherit"` | Text color (CSS value or inherit) |
| `align` | `"left" \| "center" \| "right"` | `"left"` | Text alignment |
| `balance` | `boolean` | `false` | Enable text-wrap: balance for readability |
| `animate` | `"masked-lines" \| "masked-words" \| "masked-chars" \| "scramble-load" \| "fade-up-load" \| "scramble-scroll" \| "highlight-scroll" \| false` | `false` | GSAP animation type |
| `trigger` | `"scroll" \| "load"` | `"scroll"` | Animation trigger mode |
| `delay` | `number` | `0` | Animation delay in seconds |
| `class` | `string` | `""` | Additional CSS classes |

### Data Attributes

| Attribute | Values | Purpose |
|-----------|--------|---------|
| `data-text` | Present (no value) | Component identifier |
| `data-text-animate` | Animation type or undefined | Animation selector for GSAP |
| `data-text-trigger` | "scroll", "load", or undefined | Animation trigger mode |
| `data-text-split` | "chars", "words", "lines", or undefined | SplitText split type |
| `data-text-delay` | Delay in seconds or undefined | Animation delay value |

## Accessibility

### Semantic HTML

- **Use correct heading levels** - Maintain proper document outline (h1, h2, h3, etc.)
- **Don't skip levels** - Go h2→h3, not h2→h4
- **One h1 per page** - Use h1 for main page title only

### Animation Accessibility

- **Respects prefers-reduced-motion** - Animations should be disabled for users with motion sensitivities
- **Test with screen readers** - Ensure animations don't interfere with content announcement
- **Provide text alternatives** - Don't rely solely on animation to convey meaning

### Best Practices

```html
<!-- Good: Proper heading hierarchy -->
<h1>Page Title</h1>
<h2>Section 1</h2>
<h3>Subsection 1.1</h3>
<h3>Subsection 1.2</h3>
<h2>Section 2</h2>

<!-- Good: Semantic level with visual flexibility -->
<h3 class="heading--h1">Semantic h3, displayed as h1 size</h3>

<!-- Avoid: Skipping heading levels -->
<h1>Main</h1>
<h4>Should be h2 or h3</h4>

<!-- Avoid: Multiple h1 on one page -->
<h1>Title 1</h1>
<h1>Title 2</h1>
```
