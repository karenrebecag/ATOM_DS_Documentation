## Overview

The Avatar component displays user profile images, initials, or icons with optional status badges. It's a versatile atom-level component that adapts to various contexts through size, shape, and type variants.

:::tip[Interactive Playground]
**[Open Avatar Playground →](/playground/avatar)**

View all Avatar variants with live examples including badges, shapes, and sizes in an isolated environment.
:::

**Key Features:**
- 4 size variants (xs, s, m, l)
- 4 content types (image, image-border, initials, icon)
- 2 shape options (circle, square)
- 6 badge states (none, verified, online, offline, idle, inactive)
- Fully token-driven styling
- Accessible with ARIA labels
- Auto-detection of content type

## Anatomy

The Avatar component consists of three main elements:

```
┌─────────────────────────┐
│  .avatar (container)    │  ← Base wrapper with position: relative
│  ┌──────────────────┐   │
│  │  .avatar__frame  │   │  ← Inner frame (overflow: hidden)
│  │  ┌────────────┐  │   │
│  │  │  Content   │  │   │  ← One of: __img | __initials | __icon
│  │  └────────────┘  │   │
│  └──────────────────┘   │
│                         │
│  ● .avatar__badge      │  ← Badge indicator (position: absolute)
└─────────────────────────┘
```

**DOM Structure:**
```html
<span class="avatar avatar--m avatar--circle avatar--image-border">
  <span class="avatar__frame">
    <img class="avatar__img" src="..." alt="..." />
  </span>
  <span class="avatar__badge avatar__badge--verified">
    <svg>...</svg>
  </span>
</span>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string?` | — | Image URL for avatar photo |
| `alt` | `string?` | `""` | Alt text for image (accessibility) |
| `size` | `"xs" \| "s" \| "m" \| "l"` | `"xs"` | Size variant |
| `shape` | `"circle" \| "square"` | `"circle"` | Shape variant |
| `type` | `"image" \| "image-border" \| "initials" \| "icon"` | Auto-detected | Content type |
| `initials` | `string?` | `"?"` | Initials text (e.g., "JD") |
| `badge` | `"verified" \| "online" \| "offline" \| "none"` | `"none"` | Status badge indicator |
| `class` | `string?` | — | Additional CSS classes |

**Auto-detection Logic:**
- If `src` is provided → defaults to `type="image-border"`
- If `src` is missing → defaults to `type="initials"`
- Can be overridden by explicitly passing `type`

## Variants

### Size Variants

| Size | Token | Value | Use Case |
|------|-------|-------|----------|
| **xs** | `--avatar-structure-size-xs` | `24px` | Compact lists, inline mentions |
| **s** | `--avatar-structure-size-s` | `32px` | Sidebar navigation, small cards |
| **m** | `--avatar-structure-size-m` | `40px` | Default UI elements, headers |
| **l** | `--avatar-structure-size-l` | `48px` | Profile pages, large cards |

### Type Variants

| Type | Background | Border | Description |
|------|------------|--------|-------------|
| **image** | `none` | `none` | Clean photo without decorations |
| **image-border** | `var(--bg-tertiary)` | Visual padding | Photo with subtle border effect |
| **initials** | `var(--bg-tertiary)` | — | Text-based fallback (1-2 letters) |
| **icon** | `var(--bg-tertiary)` | — | Generic user icon placeholder |

### Shape Variants

| Shape | Border Radius | Use Case |
|-------|---------------|----------|
| **circle** | `50%` or `var(--radius-pill)` | Modern, friendly UI |
| **square** | `var(--radius-s)` (xs/s) or `var(--radius-m)` (m/l) | Professional, structured layouts |

### Badge Variants

| Badge | Background Token | Icon/Indicator | Meaning |
|-------|------------------|----------------|---------|
| **none** | — | — | No status |
| **verified** | `var(--primitive-blue-500)` | checkmark SVG | Official/verified account |
| **online** | `var(--fg-accent-indicators-online)` | Solid dot | User is active |
| **offline** | `var(--fg-accent-indicators-offline)` | Solid dot | User is inactive |
| **idle** | `var(--fg-accent-indicators-idle)` | Solid dot | User is idle |
| **inactive** | `var(--fg-accent-indicators-inactive)` | Solid dot | User is inactive |

## Animations

The Avatar component itself has **no built-in animations**. It's designed as a static display element for optimal performance.

**Animation Opportunities:**
- **Entry animations** (fade-in, scale) can be applied via GSAP or CSS
- **Hover effects** for clickable avatars (scale, shadow, border glow)
- **Badge transitions** when status changes
- **Loading states** (skeleton, shimmer)

**Example with GSAP (in parent component):**
```astro
<script>
  import gsap from 'gsap';

  // Entry animation
  gsap.fromTo('.avatar',
    { scale: 0, opacity: 0 },
    { scale: 1, opacity: 1, duration: 0.4, ease: 'back.out', stagger: 0.05 }
  );
</script>
```

**Example with CSS (hover interaction):**
```css
.avatar-button {
  transition: transform 200ms ease;
}

.avatar-button:hover .avatar {
  transform: scale(1.05);
}
```

## Accessibility

### ARIA Attributes

The Avatar component includes built-in accessibility features:

```html
<span
  class="avatar ..."
  role="img"
  aria-label="John Doe"
  data-avatar
>
  <!-- Badge with aria-label -->
  <span
    class="avatar__badge avatar__badge--verified"
    aria-label="Verified"
    data-avatar-badge
  >
    <svg aria-hidden="true">...</svg>
  </span>
</span>
```

**Automatic ARIA Labels:**
- `role="img"` -- Identifies avatar as image element
- `aria-label` -- Uses `alt` prop, falls back to `initials` or "Avatar"
- Badge `aria-label` -- "Verified", "Online", "Offline"
- SVG icons use `aria-hidden="true"` to avoid duplication

### Best Practices

**Always provide meaningful alt text:**
```astro
<Avatar src="..." alt="Karen Ortiz, Design Lead" />
```

**Use initials as fallback:**
```astro
<Avatar initials="KO" alt="Karen Ortiz" />
```

**For decorative avatars, use empty alt:**
```astro
<Avatar src="..." alt="" />
```

**Don't use generic labels:**
```astro
<!-- Bad -->
<Avatar src="..." alt="User" />
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Avatar component is controlled by design tokens. This ensures:

- **Global consistency** -- All avatars use the same size scale
- **Easy theming** -- Change tokens to update all avatars
- **Responsive sizing** -- Tokens can be responsive via media queries
- **Maintainability** -- Update once, reflect everywhere

### Token Override Example

```css
/* Custom size variant (if needed) */
.avatar--xl {
  width: 64px;
  height: 64px;
}

.avatar--xl .avatar__badge {
  width: 18px;
  height: 18px;
  right: 2px;
  bottom: 2px;
}

.avatar--xl .avatar__initials {
  font-size: var(--text-body-regular);
}
```
