## Overview

The Spinner component is a loading indicator that displays continuous rotation animation to indicate ongoing async operations. It uses CSS @keyframes for optimal performance and supports 5 size variants for flexible layout integration. Ideal for form submissions, data fetching, file uploads, and any asynchronous action feedback.

**Key Features:**
- 5 size variants (xs, s, m, l, xl) for flexible layouts
- Continuous smooth rotation animation via CSS @keyframes
- Performant animation (no JavaScript overhead)
- Track + indicator dual-layer design
- Semantic status role with ARIA labels
- Data attributes for animation hooks
- Works seamlessly in buttons, loading screens, and overlays
- Respects `prefers-reduced-motion` for accessibility
- Fully token-driven styling
- Lightweight and self-contained

## Anatomy

The Spinner component has a simple two-layer structure for rotation effect:

```
┌────────────────────┐
│  <span> spinner    │  ← Container
│  ┌──────────────┐  │
│  │ <span> track │  │  ← Background track
│  │ ┌──────────┐ │  │
│  │ │ indicator│ │  │  ← Rotating indicator
│  │ └──────────┘ │  │
│  └──────────────┘  │
└────────────────────┘
```

**DOM Structure:**
```html
<span
  class="spinner spinner--m"
  role="status"
  aria-label="Loading"
  data-spinner
>
  <span class="spinner__track"></span>
  <span class="spinner__indicator" data-spinner-indicator></span>
</span>
```

**Animation:**
```css
@keyframes spinner-rotate {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

.spinner__indicator {
  animation: spinner-rotate 1s linear infinite;
}
```

## Variants

### Size Variants

| Size | Diameter | Use Case |
|------|----------|----------|
| **xs** | `16px` | Inline spinners, small indicators |
| **s** | `24px` | Form fields, compact loading |
| **m** | `32px` | Standard loading, default UI |
| **l** | `40px` | Prominent loading states |
| **xl** | `48px` | Full-page loading, hero sections |

**Size Tokens:**
```css
--spinner-size-xs: 16px;
--spinner-size-s: 24px;
--spinner-size-m: 32px;
--spinner-size-l: 40px;
--spinner-size-xl: 48px;
```

### Animation Behavior

| Property | Value | Description |
|----------|-------|-------------|
| **Duration** | `1s` | Complete rotation time |
| **Timing** | `linear` | Constant speed rotation |
| **Direction** | `infinite` | Continuous loop |
| **Motion** | `prefers-reduced-motion` | Respects accessibility preference |

## Animation Details

### CSS Keyframes

The Spinner uses a simple 360-degree rotation animation:

```css
@keyframes spinner-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

.spinner__indicator {
  animation: spinner-rotate var(--spinner-duration, 1s) linear infinite;
}
```

### Performance Characteristics

- **GPU Accelerated:** Uses `transform: rotate()` for optimal performance
- **No JavaScript:** Pure CSS animation, zero runtime overhead
- **Respects Preferences:** Automatically respects `prefers-reduced-motion`
- **Smooth:** 60fps animation on modern browsers

### Accessibility Animation

```css
@media (prefers-reduced-motion: reduce) {
  .spinner__indicator {
    animation-duration: 3s;  /* Slower for accessibility */
  }
}
```

## Accessibility

### Semantic Markup

The Spinner uses the `role="status"` to indicate dynamic content updates:

```html
<span
  class="spinner spinner--m"
  role="status"
  aria-label="Loading"
  data-spinner
>
  <span class="spinner__track"></span>
  <span class="spinner__indicator"></span>
</span>
```

### ARIA Labels

- `role="status"` — Announces spinner as status/state indicator
- `aria-label="Loading"` — Provides context for screen readers
- Spinner track marked as `aria-hidden` (decorative)

### Screen Reader Support

Screen readers announce the spinner as:
- "Loading, status" (announces the role and purpose)
- No unnecessary repetition of content
- Ideal when spinner is the sole loading indicator

### Best Practices

**Use with loading text:**
```astro
<div style="display: flex; gap: var(--s); align-items: center;">
  <Spinner size="s" />
  <span>Processing your request...</span>
</div>
```

**Use descriptive aria-label:**
```astro
<Spinner size="m" aria-label="Loading user profile..." />
```

**Combine with error states:**
```astro
{isLoading ? (
  <Spinner size="m" />
) : isError ? (
  <p role="alert">Failed to load. Please try again.</p>
) : (
  <Content />
)}
```

**Don't hide spinners from screen readers:**
```astro
<!-- Bad: Hidden from all users -->
<Spinner size="m" aria-hidden="true" />

<!-- Good: Visible and announced -->
<Spinner size="m" />
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Spinner is controlled by design tokens:

- **Global consistency** — All spinners use the same rotation speed
- **Easy theming** — Change spinner color by updating token
- **Accessibility** — Motion preferences automatically respected
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Brand-colored spinner */
.spinner--brand {
  --spinner-color-primary: var(--primitive-brand-500);
  --spinner-color-secondary: var(--primitive-brand-100);
}

/* Slower spinner for accessibility */
.spinner--slow {
  --spinner-animation-duration: 2s;
}

/* Error state spinner */
.spinner--error {
  --spinner-color-primary: var(--primitive-error-500);
}
```

**Usage:**
```astro
<Spinner size="m" class="spinner--brand" />
<Spinner size="m" class="spinner--slow" />
<Spinner size="m" class="spinner--error" />
```
