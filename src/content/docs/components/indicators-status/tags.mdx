## Overview

The Tag component is a status indicator that communicates state or status with semantic color coding. It supports 7 semantic states (success, warning, danger, info, neutral, brand, ai) and 2 visual styles (filled, outlined).

### Key Features

- **7 Semantic States**: Success, warning, danger, info, neutral, brand, and AI with distinct color coding
- **2 Visual Styles**: Filled and outlined variants for different visual hierarchies
- **Optional Dot Indicator**: Status dot for additional visual emphasis
- **Icon Support**: Slot for 16px icons alongside text
- **Avatar Support**: Slot for 24px avatars for user-specific tags
- **Lightweight**: Minimal markup with semantic `<span>` structure
- **Composable**: Works standalone or integrated with other components
- **Data-Attribute Contract**: Hooks for animations and interactions

## Installation

To use the Tag component in your project, install it from the Atom Design System package:

```bash
npm install @atomchat.io/components-astro
# or
pnpm add @atomchat.io/components-astro
```

Then import and include the CSS tokens:

```astro
---
import { Tag } from '@atomchat.io/components-astro'
import '@atomchat.io/components-astro/dist/tokens.css'
---
```

## Anatomy

The Tag component is a `<span>` wrapper that can contain a dot indicator, avatar, icon, and text:

```html
<span class="tag tag--success tag--filled" data-tag>
  <span class="tag__dot" aria-hidden="true" data-tag-dot></span>
  <span class="tag__avatar"><slot name="avatar" /></span>
  <span class="tag__icon"><slot name="icon" /></span>
  <span class="tag__text" data-tag-text>Tag Label</span>
</span>
```

**DOM Structure:**
- `.tag` - Root container with semantic styling
- `.tag--{state}` - State modifier (success, warning, danger, info, neutral, brand, ai)
- `.tag--{type}` - Type modifier (filled, outlined)
- `.tag__dot` - Optional status dot indicator (aria-hidden)
- `.tag__avatar` - Avatar container (24px)
- `.tag__icon` - Icon container (16px)
- `.tag__text` - Text content wrapper
- `data-tag` - Component identifier for animations
- `data-tag-dot` - Dot element identifier
- `data-tag-text` - Text element identifier

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `state` | `"success" \| "warning" \| "danger" \| "info" \| "neutral" \| "brand" \| "ai"` | `"success"` | Semantic state with corresponding color |
| `type` | `"filled" \| "outlined"` | `"filled"` | Visual style variant |
| `hasDots` | `boolean` | `false` | Show status dot indicator |
| `class` | `string` | `""` | Additional CSS classes |

### Slots

| Slot | Size | Description |
|------|------|-------------|
| `default` | - | Tag label text (required) |
| `icon` | 16px | Icon element before text |
| `avatar` | 24px | Avatar element before text |

### Data Attributes

| Attribute | Values | Purpose |
|-----------|--------|---------|
| `data-tag` | Present (no value) | Root element identifier |
| `data-tag-dot` | Present (no value) | Status dot identifier |
| `data-tag-text` | Present (no value) | Text content identifier |

## Variants

### By State

Tag supports 7 semantic states for different status categories:

**State Semantics:**
- **Success** (Green): Positive completion, approved, or active status
- **Warning** (Orange): Caution, pending, or needs attention
- **Danger** (Red): Error, critical, or failed status
- **Info** (Blue): Informational or neutral information
- **Neutral** (Gray): Default, inactive, or neutral state
- **Brand** (Primary): Featured, highlighted, or branded content
- **AI** (Purple): AI-generated or AI-assisted content

- `--tag-neutral-bg` / `--tag-neutral-fg` - Neutral colors
- `--tag-brand-bg` / `--tag-brand-fg` - Brand colors

**Structural Tokens:**
- `--tag-padding` - Internal spacing
- `--tag-border-radius` - Corner radius
- `--tag-font-size` - Text size
- `--tag-border-width` - Outline stroke width

## Accessibility

### ARIA Implementation

- **aria-hidden="true"** on dot indicator (purely decorative)
- Semantic text content wrapped in `.tag__text` for screen readers
- Color is not the only indicator—text labels provide context

### Best Practices

- Always use descriptive text labels with tags
- Pair tags with context or supporting text when status is critical
- Use consistent state meanings across the application
- Test with screen readers to ensure labels are announced
- Don't rely solely on color to convey meaning
- Provide alternative text for icon-only tags

### Screen Reader Testing

```html
<!-- Good: Clear text label -->
<Tag state="success">Task Completed</Tag>

<!-- Good: Text with icon -->
<Tag state="warning">
  <svg slot="icon">...</svg>
  Review Required
</Tag>

<!-- Avoid: Icon-only without context -->
<Tag state="danger">!</Tag>
```

## Design Tokens in Action

The Tag component consumes semantic color tokens organized by state:

**Color System:**
```css
/* Success state */
.tag--success.tag--filled {
  background-color: var(--tag-success-bg);
  color: var(--tag-success-fg);
}

.tag--success.tag--outlined {
  background-color: transparent;
  color: var(--tag-success-fg);
  border: var(--tag-border-width) solid var(--tag-success-fg);
}

/* Similar for warning, danger, info, neutral, brand, ai states */
```

**Shape & Structure:**
```css
.tag {
  padding: var(--tag-padding);
  border-radius: var(--tag-border-radius);
  font-size: var(--tag-font-size);
  line-height: var(--tag-line-height);
  display: inline-flex;
  align-items: center;
  gap: var(--spacing-xs);
}

.tag__dot {
  width: var(--tag-dot-size);
  height: var(--tag-dot-size);
  border-radius: 50%;
  background-color: currentColor;
}
```
