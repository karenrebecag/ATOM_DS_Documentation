## Overview

Chips are compact, interactive elements that represent selections, filters, or categories. They function as toggleable buttons with visual feedback for selected and disabled states. Ideal for filter bars, tag clouds, category selection, and attribute displays.

**Key Features:**
- 5 size variants (xs, s, m, l, xl) for flexible layouts
- Toggle selection states with visual feedback
- Disabled state support
- Leading and trailing icon slots for enhanced semantics
- Compact and self-contained design
- Semantic button HTML with type="button"
- Data attributes for animations and styling
- Full keyboard accessibility and focus management
- Fully token-driven styling
- Multi-select or single-select patterns

## Anatomy

The Chip component is a flexible, icon-aware button element:

```
┌────────────────────────────────┐
│  <button> .chip                │  ← Button wrapper
│  ┌──────────┐ ┌──────┐ ┌────┐ │
│  │ <span>   │ │      │ │    │ │
│  │ icon-    │ │label │ │icon│ │
│  │ start    │ │      │ │-end│ │
│  └──────────┘ └──────┘ └────┘ │
└────────────────────────────────┘
```

**DOM Structure (Label only):**
```html
<button
  class="chip"
  type="button"
  data-chip
  data-size="m"
  data-selected="false"
>
  <span class="chip__label" data-chip-label>
    Filter Option
  </span>
</button>
```

**DOM Structure (With icons):**
```html
<button
  class="chip"
  type="button"
  data-chip
  data-size="m"
  data-selected="true"
>
  <span class="chip__icon" data-chip-icon>
    <svg><!-- Leading icon --></svg>
  </span>
  <span class="chip__label" data-chip-label>
    Selected Filter
  </span>
  <span class="chip__icon" data-chip-icon>
    <svg><!-- Trailing icon --></svg>
  </span>
</button>
```

**DOM Structure (Disabled):**
```html
<button
  class="chip"
  type="button"
  data-chip
  data-size="m"
  data-selected="false"
  disabled
>
  <span class="chip__label" data-chip-label>
    Unavailable Filter
  </span>
</button>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Chip size variant |
| `selected` | `boolean?` | `false` | Selected state toggle |
| `disabled` | `boolean?` | `false` | Disables interaction |
| `class` | `string?` | — | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `icon-start` | Leading icon (optional) |
| `default` | Chip label text (required) |
| `icon-end` | Trailing icon (optional) |

**Slot Usage:**
```astro
<Chip size="m" selected>
  <svg slot="icon-start"><!-- Icon --></svg>
  Filter Label
  <svg slot="icon-end"><!-- Icon --></svg>
</Chip>
```

## Variants

### Size Variants

| Size | Padding | Font Size | Use Case |
|------|---------|-----------|----------|
| **xs** | `var(--chip-padding-xs)` | `11px` | Dense layouts, mobile filters |
| **s** | `var(--chip-padding-s)` | `12px` | Compact UI, secondary filters |
| **m** | `var(--chip-padding-m)` | `14px` | Standard chip, default UI |
| **l** | `var(--chip-padding-l)` | `16px` | Prominent filters, large areas |
| **xl** | `var(--chip-padding-xl)` | `18px` | Hero sections, feature highlight |

**Size Tokens:**
```css
--chip-size-xs: 28px;      /* Height for xs */
--chip-size-s: 32px;       /* Height for s */
--chip-size-m: 36px;       /* Height for m */
--chip-size-l: 40px;       /* Height for l */
--chip-size-xl: 44px;      /* Height for xl */
```

### State Variants

| State | Behavior | Visual Change |
|-------|----------|---------------|
| **Unselected** | Interactive | Default background |
| **Selected** | Active/highlighted | Different background + accent color |
| **Hover** | On mouse over | Subtle highlight transition |
| **Focus** | On keyboard focus | Focus ring outline |
| **Disabled** | Non-interactive | Gray background + `cursor: not-allowed` |
| **Disabled Selected** | Non-interactive | Grayed selected state |

## Animations

The Chip component supports optional animation effects through CSS transitions.

### Selection Animation

**What it does:**
When a chip is selected or deselected, the background color and scale animate smoothly.

**CSS-based Animation:**

```css
.chip {
  transition: all 200ms ease-out;
}

.chip[data-selected="true"] {
  background-color: var(--chip-bg-selected);
  color: var(--chip-fg-selected);
}

.chip:hover {
  transform: scale(1.05);
}

.chip:active {
  transform: scale(0.95);
}
```

### GSAP Timeline (optional)

```typescript
// Selection animation
gsap.to('[data-chip]', {
  backgroundColor: isSelected ? '#4f46e5' : '#e5e7eb',
  color: isSelected ? '#ffffff' : '#000000',
  duration: 0.2,
  ease: 'power2.out'
});
```

### Initialization

**Global initialization (optional):**

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Respects Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  .chip {
    transition-duration: 1s;
  }
}
```

## Accessibility

### Semantic HTML

The Chip component uses semantic `<button>` elements for proper keyboard interaction and screen reader support:

```html
<button
  class="chip"
  type="button"
  data-chip
  data-size="m"
  data-selected="false"
>
  <span class="chip__label">Filter Label</span>
</button>
```

### Keyboard Navigation

- **Tab/Shift+Tab:** Navigate between chips
- **Enter/Space:** Toggle chip selection
- **Disabled chips:** Not focusable, skipped in tab order

### Screen Reader Support

- Button semantic automatically announces as "button"
- Label text is read as button content
- Selected state conveyed through color/styling (ensure sufficient contrast)
- Icons should have appropriate alt text or aria-hidden

### Best Practices

**Provide clear labels:**
```astro
<Chip size="m">Filter by Category</Chip>
```

**Use aria-label for icon-only chips:**
```astro
<Chip size="m" aria-label="Remove filter">
  <svg slot="icon-end"><!-- Close icon --></svg>
</Chip>
```

**Group related chips:**
```astro
<fieldset>
  <legend>Filter by type</legend>
  <Chip size="m">Option 1</Chip>
  <Chip size="m">Option 2</Chip>
  <Chip size="m">Option 3</Chip>
</fieldset>
```

**Don't use decorative chips for interaction:**
```astro
<!-- Bad: Chip without purpose -->
<Chip size="m">View more</Chip>

<!-- Good: Use Button component instead -->
<button class="button">View more</button>
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Chip is controlled by design tokens:

- **Global consistency** — All chips use the same sizing and color scale
- **Easy theming** — Switch themes by updating token values
- **State management** — Selected/hover/focus/disabled states via tokens
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Premium badge chips */
.chip--premium {
  --chip-bg-enabled: var(--primitive-gold-100);
  --chip-bg-selected: var(--primitive-gold-600);
  --chip-border-enabled: var(--primitive-gold-300);
  --chip-fg-selected: var(--primitive-neutral-0);
}

.chip--premium:hover {
  --chip-bg-enabled: var(--primitive-gold-200);
}
```

**Usage:**
```astro
<Chip size="m" class="chip--premium">Premium Member</Chip>
```
