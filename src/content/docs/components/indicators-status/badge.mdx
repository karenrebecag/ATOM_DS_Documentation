## Overview

The Badge component is a notification indicator that displays count information with intelligent overflow handling. It automatically adjusts display values based on context-specific rules and hides when count is zero. Ideal for notification counters, unread messages, and status indicators.

**Key Features:**
- Smart overflow capping with context-aware rules
- 3 visual types (neutral, inbox, info) for different use cases
- 3 visual states (default, focused, subtle) for emphasis control
- Automatic hiding when count is zero
- Dynamic ARIA labels for screen readers
- Data attributes for animations and styling
- Flexible context rules (default: 1-99 then "99+", inbox: 1-49 then "+50")
- Semantic status role with accessibility
- Fully token-driven styling
- Compact and self-contained design

```
┌────────────────┐
│ <span> badge   │  ← Badge container
│ ┌────────────┐ │
│ │ <span>     │ │  ← Badge text (count or overflow)
│ │ .badge__   │ │
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `count` | `number` | `0` | Notification count to display |
| `context` | `"default" \| "inbox"` | `"default"` | Overflow rule context |
| `type` | `"neutral" \| "inbox" \| "info"` | `"neutral"` | Visual type variant |
| `state` | `"default" \| "focused" \| "subtle"` | `"default"` | Visual state variant |
| `class` | `string?` | — | Additional CSS classes |

### Return Value

- **Rendered:** When `count > 0`
- **Hidden (null):** When `count <= 0`

## Overflow Rules

The Badge component implements smart overflow capping based on context:

### Default Context

```
Count Input → Display Output
1-99        → Exact number (1, 5, 45, 99)
100+        → "99+"
0 or neg    → Hidden (null)
```

**Use case:** General notifications, unread counts, generic counters

**Examples:**
```astro
<Badge count={1} context="default" />        <!-- Shows: 1 -->
<Badge count={45} context="default" />       <!-- Shows: 45 -->
<Badge count={99} context="default" />       <!-- Shows: 99 -->
<Badge count={100} context="default" />      <!-- Shows: 99+ -->
<Badge count={500} context="default" />      <!-- Shows: 99+ -->
<Badge count={0} context="default" />        <!-- Hidden -->
```

### Inbox Context

```
Count Input → Display Output
1-49        → Exact number (1, 25, 49)
50+         → "+50"
0 or neg    → Hidden (null)
```

**Use case:** Email inbox, message counts, capacity-limited contexts

**Examples:**
```astro
<Badge count={1} context="inbox" />          <!-- Shows: 1 -->
<Badge count={25} context="inbox" />         <!-- Shows: 25 -->
<Badge count={49} context="inbox" />         <!-- Shows: 49 -->
<Badge count={50} context="inbox" />         <!-- Shows: +50 -->
<Badge count={500} context="inbox" />        <!-- Shows: +50 -->
<Badge count={0} context="inbox" />          <!-- Hidden -->
```

## Variants

### Type Variants

| Type | Background | Use Case |
|------|------------|----------|
| **neutral** | Gray/default color | General notifications, unread items |
| **inbox** | Primary/accent color | Email, messages, important notifications |
| **info** | Blue/info color | Information, updates, announcements |

### State Variants

| State | Visual Change | Use Case |
|-------|---------------|----------|
| **default** | Standard appearance | Normal notification display |
| **focused** | Highlighted/emphasized | Active or priority notifications |
| **subtle** | Muted/de-emphasized | Background notifications, secondary indicators |

### Context Rules

| Context | Display Rules | Use Case |
|---------|---------------|----------|
| **default** | 1-99 exact, 100+ shows "99+" | General counters, flexible capacity |
| **inbox** | 1-49 exact, 50+ shows "+50" | Email, capacity-limited contexts |

## Animations

The Badge component supports optional animation effects through the ATOM animation system.

### Badge Appearance Animation

**What it does:**
When a badge first appears or updates, it can animate with a subtle scale and fade effect.

**Data attributes:**
```html
<span class="badge" data-badge data-count="45">
  <span class="badge__text" data-badge-text>45</span>
</span>
```

**Animation Opportunities:**
- **Entrance:** Scale from 0 to 1 with fade-in
- **Update:** Pulse effect when count changes
- **Exit:** Fade-out when count becomes zero

**GSAP Timeline (optional):**

```typescript
// Entrance animation
gsap.from('[data-badge]', {
  scale: 0,
  opacity: 0,
  duration: 0.3,
  ease: 'back.out'
});

// Update pulse
gsap.to('[data-badge-text]', {
  scale: 1.2,
  duration: 0.2,
  ease: 'power2.out',
  yoyo: true,
  repeat: 1
});
```

### Initialization

**Global initialization (optional):**

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Respects Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  [data-badge] {
    animation-duration: 1s;
  }
}
```

## Accessibility

### ARIA Attributes

The Badge component includes built-in accessibility features:

```html
<!-- Standard badge -->
<span
  class="badge badge--neutral badge--default"
  role="status"
  aria-label="5 notifications"
  data-badge
  data-count="5"
>
  <span class="badge__text">5</span>
</span>

<!-- Single notification (singular form) -->
<span
  class="badge badge--inbox badge--focused"
  role="status"
  aria-label="1 notification"
  data-badge
  data-count="1"
>
  <span class="badge__text">1</span>
</span>

<!-- Overflow badge -->
<span
  class="badge badge--info badge--default"
  role="status"
  aria-label="150 notifications"
  data-badge
  data-count="150"
>
  <span class="badge__text">99+</span>
</span>

<!-- Zero count (not rendered) -->
<!-- Nothing displayed, component returns null -->
```

**Automatic behaviors:**
- `role="status"` announces badge as live region
- `aria-label` reflects actual count, not display value (accurate for screen readers)
- Dynamic singular/plural: "1 notification" vs "5 notifications"
- Badge hidden when count is zero (no unnecessary announcements)

### Best Practices

**Use semantic positioning:**
```astro
<a href="/inbox" style="position: relative;">
  <span>Inbox</span>
  <Badge count={12} type="inbox" context="inbox" />
</a>
```

**Provide context in aria-label:**
```astro
<!-- The component auto-generates descriptive labels -->
<Badge count={5} type="neutral" />
<!-- aria-label: "5 notifications" -->
```

**Don't duplicate information:**
```astro
<!-- Bad: Visual + text duplication -->
<span>
  Inbox (12)
  <Badge count={12} type="inbox" />
</span>

<!-- Good: Let badge handle the indicator -->
<span>
  Inbox
  <Badge count={12} type="inbox" />
</span>
```

**Consider context hints:**
```astro
<!-- Clear what the badge represents -->
<a href="/inbox" aria-label="Inbox">
  <Icon name="inbox" />
  <Badge count={12} type="inbox" context="inbox" />
</a>
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the Badge is controlled by design tokens:

- **Global consistency** — All badges use the same sizing and colors
- **Easy theming** — Switch themes by updating token values
- **Type/State combinations** — Different visual treatments via tokens
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Critical notification badge */
.badge--critical {
  --badge-bg-neutral: var(--primitive-red-600);
  --badge-fg-neutral: var(--primitive-neutral-0);
  --badge-border-neutral: var(--primitive-red-700);
}

/* Pulse animation for critical */
@keyframes critical-pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.1); }
}

.badge--critical {
  animation: critical-pulse 2s ease-in-out infinite;
}
```

**Usage:**
```astro
<Badge count={5} type="neutral" class="badge--critical" />
```

## Zero Count Behavior

An important feature of Badge is that it automatically hides when count is zero:

```astro
<Badge count={0} />     <!-- Returns null, nothing rendered -->
<Badge count={1} />     <!-- Shows: 1 -->
<Badge count={-5} />    <!-- Returns null, nothing rendered -->
```

This prevents empty badges from cluttering the UI and is particularly useful for dynamic counters:

```astro
---
const unreadCount = 0;
---

<!-- No badge shown when unreadCount is 0 -->
<a href="/inbox">
  Inbox
  <Badge count={unreadCount} />
</a>
```
