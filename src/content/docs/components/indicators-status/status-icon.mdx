## Overview

The Status Icon is a compact visual indicator that communicates user presence or system status. It displays four semantic states: online (green), offline (red), idle/away (yellow), and inactive (gray).

### Key Features

- **4 Status States**: Online, offline, idle/away, and inactive with semantic color coding
- **Flexible Sizing**: Four sizes (xs, s, m, l) for different contexts
- **Optional Border**: White border option to create contrast on colored backgrounds
- **Accessibility**: ARIA labels and semantic role="status" for screen readers
- **Lightweight**: Minimal markup—just a styled `<span>` element
- **Composable**: Perfect for stacking with other components like avatars

## Anatomy

The Status Icon is a single `<span>` element with semantic HTML attributes and CSS classes for styling:

```html
<span
  class="status-icon status-icon--online status-icon--xs status-icon--bordered"
  data-status-icon
  data-status="online"
  data-size="xs"
  role="status"
  aria-label="Online"
/>
```

**DOM Structure:**
- `.status-icon` - Root container (circular, applies border-radius)
- `.status-icon--{status}` - Status color modifier (online, offline, idle, inactive)
- `.status-icon--{size}` - Size modifier (xs, s, m, l)
- `.status-icon--bordered` - Optional white border
- `data-status-icon` - Component identifier
- `data-status` - Current status value
- `data-size` - Current size value
- `role="status"` - ARIA role for live region
- `aria-label` - Accessible label describing the status

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `status` | `"online" \| "offline" \| "idle" \| "inactive"` | `"online"` | The presence or status state to display |
| `size` | `"xs" \| "s" \| "m" \| "l"` | `"xs"` | The icon diameter (uses structure tokens) |
| `hasBorder` | `boolean` | `true` | Show white border around the icon |
| `class` | `string` | `""` | Additional CSS classes for customization |

### Data Attributes

| Attribute | Values |
|-----------|--------|
| `data-status-icon` | Present (no value) |
| `data-status` | "online", "offline", "idle", "inactive" |
| `data-size` | "xs", "s", "m", "l" |

## Variants

### By Status

The Status Icon provides four semantic color states representing different presence conditions:

**Status Semantics:**
- **Online** (Green): User is actively present and available
- **Offline** (Red): User is not connected or unavailable
- **Idle/Away** (Yellow): User is present but inactive
- **Inactive** (Gray): User account is disabled or inactive

### By Size

The Status Icon is available in four size options for different use cases:

**Size Scale:**
- `xs` - 8px (embedded in small components)
- `s` - 12px (avatar badges)
- `m` - 16px (standard display)
- `l` - 24px (prominent indicators)

### Border Variants

The Status Icon can display with or without a white border for contrast against different background colors:

- **With Border** (default): Creates visual separation when placed over colored backgrounds
- **No Border**: Ideal for minimal layouts or when contrast is sufficient

The border creates visual separation when placed over colored backgrounds. Remove the border for minimal layouts or when contrast is sufficient.

## Structure Tokens

The Status Icon uses these structure tokens to define sizing:

| Token | Default | Context |
|-------|---------|---------|
| `--status-icon-structure-size-xs` | 8px | Embedded badges |
| `--status-icon-structure-size-s` | 12px | Avatar corners |
| `--status-icon-structure-size-m` | 16px | Standard display |
| `--status-icon-structure-size-l` | 24px | Prominent indicators |

All sizes use these spacing and border tokens:
- `--radius-pill` - 50% border radius (full circle)
- `--stroke-xs` - White border width
- `--bg-primary` - White border color

## Dependencies

The Status Icon has no component dependencies but requires these design tokens:

**Color Tokens:**
- `--fg-accent-indicators-online` (green)
- `--fg-accent-indicators-offline` (red)
- `--fg-accent-indicators-idle` (yellow)
- `--fg-accent-indicators-inactive` (gray)

**Structural Tokens:**
- `--radius-pill` - Circular shape
- `--stroke-xs` - Border width
- `--bg-primary` - Border color

## Accessibility

### ARIA Implementation

- **role="status"**: Marks the element as a live region that announces status changes
- **aria-label**: Provides accessible descriptions:
  - "Online" for online status
  - "Away" for idle status
  - "Offline" for offline status
  - "Inactive" for inactive status

### Best Practices

- Always pair the Status Icon with text labels in data-heavy contexts (dashboards, member lists)
- Use appropriate semantic status colors consistently across the application
- Ensure sufficient color contrast when using bordered variants on dark backgrounds
- For critical status indicators (server health), supplement with text fallbacks
- Test with screen readers to ensure aria-labels are announced correctly

### Screen Reader Testing

```html
<!-- Good: Clear context with label -->
<div class="user-presence">
  <img src="user.jpg" alt="Alice Johnson" />
  <StatusIcon status="online" size="s" aria-label="Alice is online" />
</div>

<!-- Avoid: Status icon alone without context -->
<StatusIcon status="online" />
```
