## Overview

The Pagination component is a table footer row that combines a records-per-page selector, a page info label (e.g. "1–10 of 120"), and previous/next navigation buttons. It also supports a loading skeleton state.

**Key Features:**
- Left: "Records per page" label + select dropdown
- Center: "1–10 of 120" page info text
- Right: Previous / Next navigation buttons
- Loading state: `pagination--loading` shows skeleton placeholders
- Navigation buttons use secondary button token system
- Fully token-driven

## API Reference

### Modifier Classes

| Class | Description |
|-------|-------------|
| `pagination--loading` | Shows skeleton in select + center, disables nav buttons |

### Nav Button States

| State | CSS |
|-------|-----|
| Enabled | Secondary button tokens |
| Hover | `--buttons-bg-secondary-hovered` |
| Active | `--buttons-bg-secondary-pressed` |
| Disabled | Muted bg, no-drop cursor |

## Design Tokens

```css
--pagination-height
--pagination-bg-enabled
--pagination-border-width

/* Navigation buttons (reuse button tokens) */
--buttons-bg-secondary-enabled / hovered / pressed / disabled

--buttons-fg-secondary-enabled / hovered / pressed / disabled

/* Skeleton loading */
```

```html

  <div class="pagination__left">
    <span class="pagination__label">Records per page</span>
```

## Accessibility

- Navigation buttons must have `aria-label`
- Disable previous button on first page, next button on last page
- Announce page changes with `aria-live`

```html
<div class="pagination" aria-label="Table pagination">
  <!-- ... -->
  <span class="pagination__center" aria-live="polite" aria-atomic="true">
    1–10 of 120 records
  </span>
  <!-- ... -->
</div>
```
