## Overview

The **IconButton** component is an icon-only button designed for compact interfaces where space is limited or when an icon alone can clearly convey the action. It features built-in hover rotation animations and loading states, sharing the same token system as the standard Button component.

import StoryPlayground from '../../../../components/StoryPlayground.astro';

<StoryPlayground src="https://atom-ds-storybook.vercel.app/iframe.html?id=components-iconbutton--playground&viewMode=story" height={240} title="IconButton interactive playground" />

**Key Features:**
- 6 semantic variants (Primary, Secondary, Tertiary + Destructive variants)
- 5 size options (xs, s, m, l, xl)
- Hover rotation animation powered by @atomchat.io/animations
- Loading state with animated spinner
- Can render as button or link (`<a>` tag)
- Fully token-driven styling
- Accessible with required ARIA labels
- External link support with proper attributes

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"Primary"` \| `"Secondary"` \| `"Tertiary"` \| `"Destructive Primary"` \| `"Destructive Secondary"` \| `"Destructive Tertiary"` | `"Primary"` | Visual style variant |
| `size` | `"xs"` \| `"s"` \| `"m"` \| `"l"` \| `"xl"` | `"m"` | Button size |
| `href` | `string?` | вҖ” | If provided, renders as `<a>` tag instead of `<button>` |
| `label` | `string` | вҖ” | **Required** aria-label for accessibility (icon-only buttons need labels) |
| `external` | `boolean?` | `false` | Opens link in new tab (only for href). Adds `target="_blank" rel="noopener noreferrer"` |
| `disabled` | `boolean?` | `false` | Disables interaction and applies disabled styling |
| `loading` | `boolean?` | `false` | Shows loading spinner and disables interaction |
| `class` | `string?` | вҖ” | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Icon SVG content (required) |

**Slot Usage:**
```astro
<IconButton variant="Primary" size="m" label="Settings">
  <svg width="16" height="16">...</svg>
</IconButton>
```

**Notes:**
- `label` prop is **required for accessibility** вҖ” icon-only buttons need descriptive labels for screen readers
- When `href` is provided, the component renders as an `<a>` tag; otherwise as `<button type="button">`
- When `loading` is true, the icon is hidden and replaced with a spinner
- Uses the same variant/size tokens as the standard Button component

## Variants

### Size Variants

| Size | Token | Dimensions | Icon Size | Use Case |
|------|-------|------------|-----------|----------|
| **xs** | `--buttons-structure-size-xs` | `24px Г— 24px` | `12px` | Ultra-compact UI, table actions |
| **s** | `--buttons-structure-size-s` | `32px Г— 32px` | `14px` | Compact toolbars, inline actions |
| **m** | `--buttons-structure-size-m` | `40px Г— 40px` | `16px` | Default size, standard UI |
| **l** | `--buttons-structure-size-l` | `48px Г— 48px` | `20px` | Emphasis, touch targets |
| **xl** | `--buttons-structure-size-xl` | `56px Г— 56px` | `24px` | Hero actions, large touch areas |

### Visual Variants

| Variant | Background | Border | Text Color | Use Case |
|---------|------------|--------|------------|----------|
| **Primary** | `--buttons-bg-primary-enabled` | None | `--buttons-fg-primary-enabled` | Main actions (save, submit) |
| **Secondary** | `--buttons-bg-secondary-enabled` | `--buttons-border-secondary-enabled` | `--buttons-fg-secondary-enabled` | Secondary actions (cancel, edit) |
| **Tertiary** | Transparent | None | `--buttons-fg-tertiary-enabled` | Subtle actions (close, minimize) |
| **Destructive Primary** | `--buttons-bg-danger-primary-enabled` | None | `--buttons-fg-danger-primary-enabled` | Destructive primary (delete account) |
| **Destructive Secondary** | `--buttons-bg-danger-secondary-enabled` | `--buttons-border-danger-secondary-enabled` | `--buttons-fg-danger-secondary-enabled` | Destructive secondary (remove item) |
| **Destructive Tertiary** | Transparent | None | `--buttons-fg-danger-tertiary-enabled` | Subtle destructive (dismiss error) |

### State Variants

| State | Visual Change | Behavior |
|-------|---------------|----------|
| **Default** | Standard appearance | Interactive, hover animation enabled |
| **Hover** | Icon rotates, background changes | Animated via `@atomchat.io/animations` |
| **Focus** | Border + shadow (`:focus-visible`) | Keyboard focus indicator |
| **Active** | Background вҶ’ `--btn-bg-pressed` | Click/press feedback |
| **Disabled** | Reduced opacity, muted colors | Non-interactive, `disabled` attribute added |
| **Loading** | Icon hidden, spinner shown | Non-interactive, `aria-busy="true"` |

## Animations

The IconButton integrates with the **ATOM animation system** (`@atomchat.io/animations`) through a **data-attribute contract**.

### Data-Attribute Contract

The IconButton component exposes these animation hooks:

| Attribute | Purpose | Animation Module |
|-----------|---------|------------------|
| `data-icon-button` | Root element discovery | All icon button animations |
| `data-hover-rotate` | Enables hover rotation effect | `initHoverRotate()` |
| `data-icon-button-icon` | Original icon (animatable) | `initHoverRotate()` |
| `data-icon-button-icon-clone` | Clone icon (animatable) | `initHoverRotate()` |
| `data-button-spinner` | Loading spinner | Native CSS animation |

### Hover Rotation Animation

**What it does:**
On hover, the icon rotates out while a clone icon rotates in, creating a smooth flip effect.

**Animation Initialization (3 functions):**

```astro
<script>
  import { initRotateClones, initRotateCalc, initHoverRotate } from '@atomchat.io/animations/hover';

  // Initialize on first load
  document.addEventListener('DOMContentLoaded', () => {
    initRotateClones();
    initRotateCalc();
    initHoverRotate();
  });
</script>
```

**CSS Variables:**

```css
.icon-button {
  --r: 20deg;  /* Rotation angle */
  --y: 1100%;  /* Transform-origin Y (auto-calculated) */
}
```

### Loading Spinner

The loading state includes an automatic rotating spinner (CSS-based, no JS required).

**Accessibility:**
```css
@media (prefers-reduced-motion: reduce) {
  .icon-button__spinner {
    animation-duration: 1.5s; /* Slower for accessibility */
  }
}
```

## Accessibility

### ARIA Attributes

The IconButton includes built-in accessibility features:

```html
<button
  class="icon-button"
  aria-label="Settings"
  aria-disabled="false"
  aria-busy="false"
>
  <!-- Icon content -->
</button>
```

**Automatic ARIA Labels:**
- `aria-label` вҖ” **Required** descriptive label from `label` prop
- `aria-disabled="true"` вҖ” Added when `disabled` or `loading` is true
- `aria-busy="true"` вҖ” Added when `loading` is true
- `aria-hidden="true"` вҖ” Applied to icon clone (for animation)

### Best Practices

**Use semantic HTML:**
```astro
<!-- Navigation -->
<IconButton variant="Secondary" href="/settings" label="Settings">
  <svg>...</svg>
</IconButton>

<!-- Action -->
<IconButton variant="Primary" label="Save changes">
  <svg>...</svg>
</IconButton>
```

**Provide descriptive labels:**
```astro
<!-- Good -->
<IconButton label="Delete item">
  <svg>...</svg>
</IconButton>

<!-- Bad -->
<IconButton label="Icon">
  <svg>...</svg>
</IconButton>
```

**Provide context in labels:**
```astro
<!-- Good -->
<IconButton label="Delete user account">
  <svg>...</svg>
</IconButton>

<!-- Bad -->
<IconButton label="Delete">
  <svg>...</svg>
</IconButton>
```

**Don't omit labels (inaccessible):**
```astro
<!-- Very Bad - Screen readers can't describe this -->
<IconButton>
  <svg>...</svg>
</IconButton>
```

### Keyboard Navigation

- **Tab**: Focus the button
- **Enter** / **Space**: Activate the button
- **Disabled/Loading**: Non-focusable, non-interactive

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the IconButton is controlled by design tokens. This ensures:

- **Global consistency** вҖ” All icon buttons use the same scale
- **Easy theming** вҖ” Change tokens to update all instances
- **Responsive sizing** вҖ” Tokens can be responsive via media queries
- **Maintainability** вҖ” Update once, reflect everywhere

### Token Override Example

```css
/* Custom XXL size variant */
.icon-button--xxl {
  width: 64px;
  height: 64px;
  border-radius: var(--radius-lg);
}

.icon-button--xxl svg {
  width: 32px;
  height: 32px;
}
```
