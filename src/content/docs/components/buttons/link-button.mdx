## Overview

The LinkButton component renders semantic `<a>` elements styled as buttons, providing a consistent navigation experience throughout your application. It supports the same visual variants and sizes as the Button component, making it ideal for primary navigation, breadcrumbs, pagination, and action links.

**Key Features:**
- 6 visual variants (Primary, Secondary, Tertiary, Destructive variants)
- 5 size options (xs, s, m, l, xl)
- Semantic HTML `<a>` tags for SEO and accessibility
- Icon support (left, right, or both)
- Hover rotation animation with GSAP
- Disabled state styling (via class)
- Fully token-driven styling
- Accessible with ARIA labels and semantic HTML

## Anatomy

The LinkButton component is structurally similar to the Button component but always renders as an `<a>` element:

```
┌──────────────────────────┐
│  .link-button (link)     │  ← Base <a> element
│  ┌────────────────────┐  │
│  │ .link-button__label-wrap│  ← Grid for hover rotation animation
│  │  ┌──────────────┐  │  │
│  │  │ __label      │  │  │  ← Original label (visible)
│  │  │ __label--clone│ │  │  ← Cloned label (hidden, for animation)
│  │  └──────────────┘  │  │
│  └────────────────────┘  │
└──────────────────────────┘
```

**DOM Structure:**
```html
<a class="link-button link-button--primary link-button--m" href="/about">
  <span class="link-button__label-wrap">
    <span class="link-button__label" data-button-label>
      Learn More
    </span>
    <span class="link-button__label link-button__label--clone" data-button-label-clone aria-hidden="true">
      Learn More
    </span>
  </span>
</a>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `href` | `string` | — | Navigation URL (required) |
| `variant` | `"Primary" \| "Secondary" \| "Tertiary" \| "Destructive Primary" \| "Destructive Secondary" \| "Destructive Tertiary"` | `"Primary"` | Visual style variant |
| `size` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Size of the link |
| `target` | `"_self" \| "_blank" \| "_parent" \| "_top"` | `"_self"` | Link target attribute |
| `rel` | `string?` | — | Relationship attribute (e.g., `"noopener noreferrer"` for `target="_blank"`) |
| `disabled` | `boolean?` | `false` | Disables link styling and interaction (applies `link-button--disabled` class) |
| `class` | `string?` | — | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Link label text (required) |
| `icon-left` | Icon placed before the label |
| `icon-right` | Icon placed after the label |

**Slot Usage:**
```astro
<LinkButton variant="Primary" size="m" href="/about">
  <svg slot="icon-left">...</svg>
  Learn More
  <svg slot="icon-right">...</svg>
</LinkButton>
```

## Variants

### Size Variants

| Size | Padding (Y × X) | Font Size | Use Case |
|------|-----------------|-----------|----------|
| **xs** | `var(--xxs) × var(--xs)` | `--text-label-regular` | Compact UIs, inline actions, breadcrumbs |
| **s** | `var(--xs) × var(--s)` | `--text-label-regular` | Dense layouts, toolbars, secondary nav |
| **m** | `var(--sm) × var(--m)` | `--text-label-regular` | Default link size, primary navigation |
| **l** | `var(--m) × var(--l)` | `--text-body-regular` | Prominent navigation links |
| **xl** | `var(--l) × var(--xl)` | `--text-body-large` | Hero sections, CTAs, main navigation |

### Visual Variants

| Variant | Background | Border | Text Color | Use Case |
|---------|------------|--------|------------|----------|
| **Primary** | `--link-button-bg-primary-enabled` | None | `--link-button-fg-primary-enabled` | Main navigation links |
| **Secondary** | `--link-button-bg-secondary-enabled` | `--link-button-border-secondary-enabled` | `--link-button-fg-secondary-enabled` | Secondary navigation |
| **Tertiary** | `--link-button-bg-tertiary-enabled` | None | `--link-button-fg-tertiary-enabled` | Subtle navigation, breadcrumbs |
| **Destructive Primary** | `--link-button-bg-danger-primary-enabled` | None | `--link-button-fg-danger-primary-enabled` | Destructive links (unsubscribe, delete account) |
| **Destructive Secondary** | `--link-button-bg-danger-secondary-enabled` | `--link-button-border-danger-secondary-enabled` | `--link-button-fg-danger-secondary-enabled` | Secondary destructive links |
| **Destructive Tertiary** | `--link-button-bg-danger-tertiary-enabled` | None | `--link-button-fg-danger-tertiary-enabled` | Subtle destructive links |

### State Variants

| State | Behavior | Visual Change |
|-------|----------|---------------|
| **Default** | Clickable, interactive | Standard appearance |
| **Hover** | On mouse over | Background → `--link-button-bg-hover`, text rotates |
| **Focus** | On keyboard focus | Border + shadow (`:focus-visible`) |
| **Active** | Currently active page | Background → `--link-button-bg-active` |
| **Disabled** | Non-interactive | Gray background + `cursor: not-allowed` |
| **Visited** | After navigation | Optional color change via `--link-button-fg-visited` |

## Animations

The LinkButton component integrates with the **ATOM animation system** (`@atomchat.io/animations`) for **hover rotation** effects.

### Hover Rotation Animation

**What it does:**
On hover, the link label rotates out while a clone label rotates in, creating a smooth flip effect.

**Data attributes:**
```html
<a class="link-button" data-button data-hover-rotate href="/about">
  <span class="link-button__label-wrap">
    <span class="link-button__label" data-button-label>
      Learn More
    </span>
    <span class="link-button__label link-button__label--clone" data-button-label-clone aria-hidden="true">
      Learn More
    </span>
  </span>
</a>
```

**Customization:**

```css
/* Override rotation angle */
.link-button--subtle-rotate {
  --r: 10deg;
}

/* Disable hover animation for specific links */
.link-button--no-animation {
  --r: 0deg;
}
```

### Initialization

**Global initialization (recommended):**

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Respects Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  .link-button {
    --r: 0deg; /* No rotation on reduced motion */
  }
}
```

## Accessibility

### ARIA Attributes

The LinkButton component includes built-in accessibility features:

```html
<!-- Standard navigation link -->
<a
  class="link-button link-button--primary"
  href="/about"
>
  Learn More
</a>

<!-- Disabled link (prevent navigation) -->
<a
  class="link-button link-button--disabled"
  aria-disabled="true"
  role="link"
>
  Coming Soon
</a>

<!-- External link with proper labeling -->
<a
  class="link-button link-button--secondary"
  href="https://example.com"
  target="_blank"
  rel="noopener noreferrer"
  aria-label="Visit Example (opens in new window)"
>
  <svg slot="icon-right"><!-- External icon --></svg>
  Learn More
</a>

<!-- Active navigation indicator -->
<a
  class="link-button link-button--primary link-button--active"
  href="/about"
  aria-current="page"
>
  About
</a>
```

**Automatic behaviors:**
- Semantic `<a>` tags for proper navigation
- `aria-disabled="true"` for disabled links
- Clone label has `aria-hidden="true"` to avoid duplication
- `aria-current="page"` for active navigation
- `rel="noopener noreferrer"` for external links with `target="_blank"`

### Best Practices

**Use semantic HTML for navigation:**
```astro
<nav>
  <LinkButton variant="Primary" size="m" href="/about">
    About
  </LinkButton>
  <LinkButton variant="Primary" size="m" href="/services">
    Services
  </LinkButton>
</nav>
```

**Provide descriptive labels:**
```astro
<LinkButton variant="Secondary" href="https://example.com" target="_blank">
  <svg slot="icon-right"><!-- External --></svg>
  Visit Example (opens in new window)
</LinkButton>
```

**Mark active navigation with `aria-current`:**
```astro
<LinkButton
  variant="Primary"
  href="/current-page"
  aria-current="page"
  class="link-button--active"
>
  Current Page
</LinkButton>
```

**Use proper `rel` attribute for external links:**
```astro
<LinkButton
  variant="Secondary"
  href="https://external.com"
  target="_blank"
  rel="noopener noreferrer"
>
  External Site
</LinkButton>
```

**Don't use generic link text:**
```astro
<!-- Bad -->
<LinkButton variant="Primary" href="/docs">
  Click Here
</LinkButton>
```

**Don't nest interactive elements:**
```astro
<!-- Bad -->
<LinkButton variant="Primary" href="/link">
  <button>Button inside link</button>
</LinkButton>
```

**Don't open new tabs without warning:**
```astro
<!-- Bad -->
<LinkButton variant="Secondary" href="https://example.com" target="_blank">
  Visit Example
</LinkButton>
```

## Design Tokens in Action

### Token-First Philosophy

Every visual property of the LinkButton component is controlled by design tokens. This ensures:

- **Global consistency** — All links use the same size and color scale
- **Easy theming** — Change tokens to update all navigation
- **Visited state management** — Track user journey via tokens
- **Maintainability** — Update once, reflect everywhere

### Token Override Example

```css
/* Custom navigation variant */
.link-button--nav {
  --link-button-bg-primary-enabled: transparent;
  --link-button-fg-primary-enabled: var(--primitive-neutral-600);
  --link-button-bg-primary-hovered: var(--primitive-neutral-50);
  --link-button-border-primary-focused: var(--primitive-primary-500);
}
```

**Usage:**
```astro
<LinkButton variant="Primary" class="link-button--nav" href="/about">
  About
</LinkButton>
```
