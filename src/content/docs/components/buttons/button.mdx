## Overview

The Button component is the primary interactive element used to trigger actions throughout the application. It supports 6 visual variants, 5 sizes, and multiple states including loading and disabled.

import StoryPlayground from '../../../../components/StoryPlayground.astro';

<StoryPlayground src="https://atom-ds-storybook.vercel.app/iframe.html?id=components-button--playground&viewMode=story" />

**Key Features:**
- 6 visual variants (Primary, Secondary, Tertiary, Destructive variants)
- 5 size options (xs, s, m, l, xl)
- Loading state with spinner animation
- Icon support (left, right, or both)
- Disabled state handling
- Can render as button or link (`<a>` tag)
- GSAP hover rotation animation
- Fully token-driven styling
- Accessible with ARIA attributes

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"Primary" \| "Secondary" \| "Tertiary" \| "Destructive Primary" \| "Destructive Secondary" \| "Destructive Tertiary"` | `"Primary"` | Visual style variant |
| `size` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Size of the button |
| `href` | `string?` | — | If provided, renders as `<a>` tag instead of `<button>` |
| `type` | `"button" \| "submit" \| "reset"` | `"button"` | Button type attribute (only for `<button>` elements) |
| `disabled` | `boolean?` | `false` | Disables button interaction |
| `loading` | `boolean?` | `false` | Shows loading spinner and disables interaction |
| `class` | `string?` | — | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `default` | Button label text (required) |
| `icon-left` | Icon placed before the label |
| `icon-right` | Icon placed after the label |


### Anatomy

## Animations

The Button component integrates with the **ATOM animation system** (`@atomchat.io/animations`) through a **data-attribute contract** that maintains separation of concerns between components and animations.

### Animation Architecture

**Separation of Concerns:**

Following the ATOM Design System architecture, Button components **do not import or initialize animations**. Instead:

1. **Component Layer** — Button.astro renders semantic HTML with `data-*` attributes
2. **Animation Layer** — GSAP functions discover elements via `querySelector("[data-*]")` and attach behaviors
3. **Layout Initialization** — Animations initialize globally in the layout, not in components

**Why this matters:**
- Zero framework coupling
- Components remain server-renderable
- Animations work across all frameworks (Astro, React, Vue, Angular)
- Easy to disable animations globally without touching components

### Data-Attribute Contract

The Button component exposes these animation hooks:

| Attribute | Purpose | Animation Module |
|-----------|---------|------------------|
| `data-button` | Root element discovery | All button animations |
| `data-hover-rotate` | Enables hover rotation effect | `initHoverRotate()` |
| `data-button-label` | Original label (animatable) | `initHoverRotate()` |
| `data-button-label-clone` | Clone label (animatable) | `initHoverRotate()` |
| `data-shimmer` | Loading text shimmer | `initShimmerText()` |
| `data-button-spinner` | Loading spinner | Native CSS animation |

### Hover Rotation Animation

**What it does:**
On hover, the button label rotates out while a clone label rotates in, creating a smooth flip effect.

**Pre-Rendered DOM Structure:**

The Button component **pre-renders both the original and clone labels** in the template:

```html
<button class="button" data-button data-hover-rotate>
  <span class="button__label-wrap">
    <!-- Original label (visible, opacity: 1, rotation: 0deg) -->
    <span class="button__label" data-button-label>
      <span class="button__icon"><!-- icon-left slot --></span>
      Button Text
      <span class="button__icon"><!-- icon-right slot --></span>
    </span>

    <!-- Clone label (hidden, opacity: 0, rotation: -20deg) -->
    <span class="button__label button__label--clone"
          data-button-label-clone
          aria-hidden="true">
      <span class="button__icon"><!-- icon-left slot --></span>
      Button Text
      <span class="button__icon"><!-- icon-right slot --></span>
    </span>
  </span>
</button>
```

**Animation Initialization (3 functions):**

The `@atomchat.io/animations` package runs three coordinated functions:

1. **`initRotateClones()`**
   Detects pre-rendered clones (Astro pattern) and marks elements with `data-clones-ready` attribute. For non-Astro contexts, creates clones inline-styled as fallback.

2. **`initRotateCalc()`**
   Calculates `--y` CSS custom property (transform-origin Y-coordinate) based on character count:
   ```typescript
   const y = Y_MIN + Y_PADDING * (Y_BASE_OFFSET + Y_CHAR_MULTIPLIER * chars);
   element.style.setProperty('--y', y + '%');
   ```
   Uses `MutationObserver` to recalculate on text changes.

3. **`initHoverRotate()`**
   Registers GSAP timelines on `pointerenter` events:
   - Reads `--r` CSS variable (default: `20deg`) for rotation angle
   - Applies "resetToRest" pattern: forces originals to `0deg` and clones to `-r` before animating
   - On hover: originals rotate `+=${r}` and fade out, clones rotate to `0deg` and fade in
   - Duration: `DURATION.half` (0.3s), stagger: `0.075s`
   - Respects `prefersReducedMotion()` and `isMotionExempt()`

**GSAP Timeline (actual implementation):**

```typescript
// resetToRest pattern (prevents state corruption on rapid re-hover)
originals.forEach(orig => gsap.set(orig, { rotation: 0, opacity: 1 }));
clones.forEach(clone => gsap.set(clone, { rotation: -r, opacity: 0 }));

// Hover animation
const tl = gsap.timeline();

tl.to(originals, {
  rotation: `+=${r}`,        // Rotate out
  opacity: 0,                // Fade out
  duration: DURATION.half,   // 0.3s
  ease: "atom",              // Custom ease: cubic-bezier(0.625, 0.05, 0, 1)
  stagger: 0.075,            // Stagger per child span
}, 0);

tl.to(clones, {
  rotation: 0,               // Rotate from -r to 0
  opacity: 1,                // Fade in
  duration: DURATION.half,
  ease: "atom",
  stagger: 0.075,
}, 0);
```

**CSS Variables:**


**Implementation:**

### Initialization

**Global initialization (recommended):**

In your Astro layout or React app entry:

```astro
<!-- Astro Layout.astro -->
<script>
  import { initAllAnimations } from '@atomchat.io/animations';

  // Initialize on first load
  document.addEventListener('DOMContentLoaded', () => {
    initAllAnimations();
  });

  // Reinitialize after Astro page transitions
  document.addEventListener('astro:page-load', () => {
    initAllAnimations();
  });
</script>
```

**Scoped initialization (SPAs, component-level):**

```typescript
import { initHoverRotate, initRotateClones, initRotateCalc } from '@atomchat.io/animations';

// In React/Vue/Svelte component
useEffect(() => {
  const cleanups = [
    initRotateClones({ scope: containerRef.current }),
    initRotateCalc({ scope: containerRef.current }),
    initHoverRotate({ scope: containerRef.current }),
  ];

  return () => cleanups.forEach(cleanup => cleanup());
}, []);
```

**Motion preferences:**

The animation system respects three-tier motion control:

1. **OS preference:** `prefers-reduced-motion: reduce`
2. **Site-level:** `<html data-motion="reduced|none">`
3. **Element-level:** `data-motion-exempt` attribute

When motion is reduced, hover rotation animations are skipped entirely inside event handlers.

## Accessibility

### ARIA Attributes

The Button component includes built-in accessibility features:

```html
<!-- Disabled button -->
<button
  class="button button--disabled"
  disabled
  aria-disabled="true"
>
  Click me
</button>

<!-- Loading button -->
<button
  class="button button--loading"
  disabled
  aria-disabled="true"
  aria-busy="true"
>
  Loading
</button>

<!-- Link button -->
<a
  class="button button--primary"
  href="/about"
>
  Learn More
</a>
```

**Automatic behaviors:**
- `disabled` attribute prevents interaction
- `aria-disabled="true"` for screen readers
- `aria-busy="true"` during loading state
- Clone label has `aria-hidden="true"` to avoid duplication
- Spinner has `aria-hidden="true"` (state conveyed by `aria-busy`)

### Best Practices

**Use semantic HTML:**
```astro
<!-- Form submission -->
<Button variant="Primary" type="submit">Submit</Button>

<!-- Navigation -->
<Button variant="Secondary" href="/about">About</Button>
```

**Provide descriptive labels:**
```astro
<Button variant="Destructive Primary">
  Delete Account
</Button>
```

**Use loading state for async actions:**
```astro
<Button variant="Primary" loading={isSubmitting}>
  {isSubmitting ? 'Saving...' : 'Save Changes'}
</Button>
```

**Don't use generic labels:**
```astro
<!-- Bad -->
<Button variant="Primary">Click Here</Button>
```

**Don't nest interactive elements:**
```astro
<!-- Bad -->
<Button variant="Primary">
  <a href="/link">Link inside button</a>
</Button>
```

### Shared Animations

All frameworks use the same animation code (optional, from `@atomchat.io/animations`):

```typescript title="hover.ts"
import { gsap, DURATION } from './config';
import { prefersReducedMotion, isMotionExempt } from './motion';

const LABEL_SELECTORS = '.button__label';

/**
 * Detect pre-rendered clones in components.
 * Marks elements as ready for animation.
 */
export function initRotateClones() {
  const elements = document.querySelectorAll('[data-hover-rotate]');
  if (!elements.length) return () => {};

  elements.forEach((el) => {
    if (el.hasAttribute('data-clones-ready')) return;

    // Check if clones are pre-rendered
    const hasPreRenderedClones = el.querySelector('[aria-hidden="true"]');
    if (hasPreRenderedClones) {
      el.setAttribute('data-clones-ready', '');
      return;
    }

    // Fallback: create clones dynamically
    const items = el.querySelectorAll(LABEL_SELECTORS);
    const r = parseFloat(getComputedStyle(el).getPropertyValue('--r')) || 20;

    items.forEach((item) => {
      const clone = item.cloneNode(true);
      clone.setAttribute('aria-hidden', 'true');
      clone.style.position = 'absolute';
      clone.style.transform = `rotate(${-r}deg)`;
      clone.style.opacity = '0';
      item.parentElement.appendChild(clone);
    });

    el.setAttribute('data-clones-ready', '');
  });

  return () => {};
}

/**
 * Calculate transform-origin Y for rotation.
 */
export function initRotateCalc() {
  const elements = document.querySelectorAll('[data-hover-rotate]');
  if (!elements.length) return () => {};

  function update(el) {
    const labels = el.querySelectorAll(LABEL_SELECTORS);
    let maxChars = 0;
    labels.forEach((label) => {
      if (label.getAttribute('aria-hidden') === 'true') return;
      const len = (label.textContent || '').trim().length;
      if (len > maxChars) maxChars = len;
    });

    const y = Math.max(100, Math.min(10000, 100 + 30 * (12 + 6 * maxChars)));
    el.style.setProperty('--y', y + '%');
  }

  elements.forEach((el) => update(el));

  return () => {};
}

/**
 * Execute rotation animation on hover.
 */
export function initHoverRotate() {
  const elements = document.querySelectorAll('[data-hover-rotate]');
  if (!elements.length) return () => {};

  const listeners = [];

  elements.forEach((el) => {
    const onEnter = () => {
      if (prefersReducedMotion() || isMotionExempt(el)) return;

      const r = parseFloat(getComputedStyle(el).getPropertyValue('--r')) || 20;
      const items = Array.from(el.querySelectorAll(LABEL_SELECTORS));

      const originals = items.filter(item => item.getAttribute('aria-hidden') !== 'true');
      const clones = items.filter(item => item.getAttribute('aria-hidden') === 'true');

      // Reset to rest state
      originals.forEach(orig => gsap.set(orig, { rotation: 0, opacity: 1 }));
      clones.forEach(clone => gsap.set(clone, { rotation: -r, opacity: 0 }));

      // Animate
      const tl = gsap.timeline();
      tl.to(originals, { rotation: `+=${r}`, opacity: 0, duration: 0.3, ease: 'power2.out', stagger: 0.075 }, 0);
      tl.to(clones, { rotation: 0, opacity: 1, duration: 0.3, ease: 'power2.out', stagger: 0.075 }, 0);
    };

    el.addEventListener('pointerenter', onEnter);
    listeners.push({ el, handler: onEnter });
  });

  return () => {
    listeners.forEach(({ el, handler }) => {
      el.removeEventListener('pointerenter', handler);
    });
  };
}
```
